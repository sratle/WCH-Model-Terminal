# Power-1 模块（供电管理）

## 概述

本模块为核心底板及所有外设提供稳定 5V 供电，同时承担电池管理、充放电控制及功耗监测功能。供电管理芯片采用 IP5568，负责 PD 快充双向协议、无线充电、充放电管理以及 LED 电量指示驱动。CH32V103C8T6 不直接参与电源路径控制，仅通过解析 IP5568 驱动的七段 LED 刷新时序来间接获取当前电池电量，并将电量信息通过 UART 上报给核心模块。

- **主控芯片**：CH32V103C8T6
- **电源管理芯片**：IP5568（支持双向 PD 快充、无线充电、5V 稳压输出、LED 电量指示）
- **电池**：6000mAh 锂电池
- **与核心模块接口**：模块侧 UART1，**921600/8-N-1**
- **模块 ID**：`0x30`
- **模块类型编号**：`0x03`
- **子类型编号**：`0x01`（标准供电模块）

## 硬件组成

- **CH32V103C8T6**：电量信息解析、UART 通信
- **IP5568**：电源管理核心芯片，负责 PD 快充协议握手、无线充电收发、电池充放电管理、5V 稳压输出以及七段 LED 电量指示驱动
- **电池**：6000mAh 锂电池
- **无线充电线圈与接收电路**（若支持无线充电）
- **USB-C 接口**：PD 快充输入 / 输出
- **七段 LED 电量指示**：由 IP5568 驱动，CH32V103C8T6 通过解析其刷新时序获取电量

## 软件职责

解析 IP5568 驱动的七段 LED 刷新时序以获取当前电池电量，并通过 UART1 上报给核心模块（CH32H417）。CH32V103C8T6 不直接控制 PD 快充、无线充电或充放电过程，这些功能由 IP5568 硬件独立完成。

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`battery.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Battery_Get_Soc()`
- **变量**：统一小写加下划线，例如 `bat_voltage`、`charge_status`
- **文件夹**：下划线写法，仅首字母大写，例如 `Battery`、`Charger`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **安全第一**：电池管理涉及过充、过放、过流、过热等风险，硬件层面必须有多重保护（保护板 / PCM），软件层面做二次监测与告警。
2. **电量精度**：若使用 ADC 分压测电压估算 SOC，需根据电池放电曲线做多点校准；若使用专用电量计芯片，需按手册配置电池化学 ID 与设计容量。
3. **PD 快充协议**：PD 握手由专用芯片完成时，CH32V103 只需通过 I2C 读取芯片状态；若需软件实现 PD 物理层，工作量极大，不建议。
4. **对外慢充输出**：作为充电宝对外输出 5V 时，需限流保护（建议 1A 以内），防止电池过放或过热。
5. **低功耗状态**：Power 模块是整机电源源头，即使系统休眠也需保持最小监测功能；CH32V103 可在浅睡模式下定时唤醒采样，降低待机电流。
6. **与 Core 的协同**：Core 侧根据电量决定性能策略（如低电量时降频、关闭背光、暂停非必要模块），Power 模块只负责如实上报数据，策略决策在 Core。
