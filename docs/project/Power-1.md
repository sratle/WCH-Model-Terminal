# Power-1 模块（供电管理）

## 概述

本模块为核心底板及所有外设提供稳定 5V 供电，同时承担电池管理、充放电控制及功耗监测功能。供电管理由 **IP5568** 芯片独立完成（PD 快充双向协议、无线充电、充放电管理、5V 稳压输出、数码管电量指示驱动）。**CH32V103C8T6 不直接参与电源路径控制**，仅：

1. 回复 Core 的 `CMD_GET_TYPE` 心跳，证明 Power 模块已接入；
2. 解析 IP5568 驱动电量数码管（188 型 YF2252SR-5，SMG1~SMG5）的信号，换算为 `0~100` 电量百分比与**充电状态**，并在其变化时通过 UART 上报 Core。

- **主控芯片**：CH32V103C8T6
- **电源管理芯片**：IP5568（双向 PD 快充、无线充电、5V 稳压输出、数码管电量指示）
- **电池**：6000mAh 锂电池
- **与核心模块接口**：模块侧 **USART1**，**230400 / 8-N-1**（对接 Core 的 UART5）
- **模块 ID**：`0x30`　**模块类型编号**：`0x03`　**子类型编号**：`0x01`（标准供电模块）

> 受硬件限制，Power 模块仅上报**电量 + 充电状态**（充电状态由数码管闪烁频率推断），不提供电压/电流/温度/功率、充电策略、告警等能力。协议见 `Protocol_Power.md`（V2.1）。

## 硬件组成与接线

| 项目 | 说明 |
|------|------|
| CH32V103C8T6 | 电量解析 + UART 通信 |
| IP5568 | 电源管理核心：PD 快充、无线充、充放电、5V 输出、数码管电量指示驱动 |
| 电量数码管 | 查理复用(Charlieplexing) 2 位半（188 型 YF2252SR-5，5 脚），显示真实 0~100，由 IP5568 SMG1~SMG5 驱动 |

**关键接线**：

| CH32V103 引脚 | 连接 | 用途 |
|---------------|------|------|
| PB3 | IP5568 SMG1 → 数码管 PIN1 | 查理复用脚1（PB3=JTDO，需关 SWJ 作 GPIO） |
| PB4 | IP5568 SMG2 → 数码管 PIN2 | 查理复用脚2（PB4=NJTRST，同上） |
| PB5 | IP5568 SMG3 → 数码管 PIN3 | 查理复用脚3 |
| PB6 | IP5568 SMG4 → 数码管 PIN4 | 查理复用脚4 |
| PB7 | IP5568 SMG5 → 数码管 PIN5 | 查理复用脚5 |
| PA9 (USART1_TX) / PA10 (USART1_RX) | Core UART5 | 协议通信 |

> 固件通过 `GPIO_Remap_SWJ_Disable` 释放 PB3/PB4（CH32V103 库仅提供整体 SWJ 关闭，会同时关闭调试口；需调试时临时注掉该行）。

## 软件职责

1. **心跳即时应答**：`GET_TYPE`(0x01) 在 **USART1 接收中断里当场回** `CMD_ACK`(0x04)+身份，应答延迟 <1ms，**不受主循环数码管采样阻塞影响**（修复之前因轮询应答导致的心跳高频 miss/误判离线）。中断是**唯一 TX 点**，每次心跳顺带上报当前电量/充电，避免与主循环抢 UART。
2. **电量/充电上报**：主循环非阻塞地解析数码管 → 真实电量 0~100 + 充电状态，更新到 `s_reported_pct/charge`；由心跳中断携带 `CMD_PWR_STATUS_REPORT`(0x36)，DATA=`[电量:1][充电:1]`，Core 侧变化才推 Display。

CH32V103 不控制 PD 快充、无线充电或充放电过程，这些由 IP5568 硬件独立完成。

## 数码管电量解码（查理复用 0~100）

188 是 **5 脚查理复用(Charlieplexing) 2 位半数显**，显示真实百分比数字（非“段式条”）。每个段是一颗 LED，接在某“高电平脚(anode)”与“低电平脚(cathode)”之间；IP5568 逐时隙把一个脚拉高、若干脚拉低、其余高阻，快速轮扫拼出数字。

**数据手册图3 段码表**（anode→cathode，脚号 1~5），已内置为 `s_cp_map[]`：

| 位 | 段 → (高脚→低脚) |
|----|------|
| 百位“1” | B1=3→4  C1=2→4 |
| 十位 | A2=2→3 B2=3→2 C2=4→3 D2=4→2 E2=5→2 F2=5→3 G2=5→4 |
| 个位 | A3=1→2 B3=2→1 C3=1→3 D3=3→1 E3=1→4 F3=4→1 G3=5→1 |

**普通 GPIO 旁路解码（双配置法）**：查理复用靠三态，普通 IO 只有高/低两态，无法直接分辨“驱动低”与“Hi-Z”。利用内部上/下拉在**同一扫描时隙**内先后两次读取来分离：

1. **内部下拉(IPD)读**：被驱动高的脚=1，其余(Hi-Z/驱动低)=0 → 得到 **anode**；
2. **内部上拉(IPU)读**：被驱动低的脚=0，其余(Hi-Z/anode)=1 → 得到 **cathode 集**；
3. 两次读相隔仅数十 µs（远小于 ms 级扫描时隙），即可配对出 (anode→cathode) → 点亮的段；
4. 多时隙累计后按 `CP_LIT_MIN` 阈值定“点亮”，再按 7 段字模解出十位/个位数字，百位由 B1&C1 判定 → 拼成 0~100。

> CFGLR 中 PB3~PB7 固定为“输入带上下拉”，切换仅写 OUTDR（上拉/下拉），极快。关键参数：`CP_SETTLE_US`(上/下拉建立时间)、`CP_GAP_US`、`CP_SAMPLES`、`CP_LIT_MIN`，均可按实测扫描速率调整。

> ⚙️ 脚位映射假设 **数码管PIN k = SMG k = PB(k+2)**；若实际顺序不同，改 `PIN_BIT()` 或接线即可。若扫描过快导致同时隙两读跨时隙，可适当减小建立时间或外接上/下拉。

### 充电状态检测（依据数据手册闪烁规律，与数字解码正交）

IP5568 188 型数码管以**整屏闪烁频率**区分充/放电：

| 显示 | 含义 | 上报 |
|------|------|------|
| 0.5Hz 闪烁（周期约 2s） | 充电中（未充满 0~99%） | 充电=1，电量=解码值 |
| 常亮 100% | 已充满 | 充电=0，电量=100 |
| 1Hz 闪烁（周期约 1s） | 放电且电量 <5% | 充电=0，电量≈5 |
| 常亮（非满） | 放电 5~100% | 充电=0，电量=解码值 |

主循环每 ≈ `LOOP_MS`(20ms) 用 `Power_Charlie_Active()` 快速采样“整屏是否点亮”，记录亮灭跳变半周期：≈**1000ms→充电**、≈**500ms→放电<5%**、>`BLINK_STEADY_MS`(2.5s) 无跳变→**常亮**。数字重建用 `Power_Charlie_Step()` **分块**到多圈累计（每圈 `CP_BATCH` 个采样，单圈不阻塞），仅在亮相阶段进行；且个位空白/乱码的本轮结果会被丢弃（避免熄灭相误读 0）。

## 固件结构

当前实现为**最小自包含**：核心逻辑集中在 `User/main.c`：

- `USART1_IRQHandler` / `Power_RxParse`：接收中断 + 逐字节协议状态机（帧头 `0xAA`、帧尾 `A5 5A FC FD`）；解出完整帧后**在中断内即时**应答 GET_TYPE。
- `Power_UART_Init` / `Power_SendGetTypeAck` / `Power_SendStatus`：UART @230400 初始化与打包发送（仅由中断调用）。
- `Power_SMG_Init` / `Power_Charlie_Active` / `Power_Charlie_Begin` / `Power_Charlie_Step` / `Power_Charlie_Decode` / `cp_decode_digit` / `Power_Detector_*` / `Power_Evaluate`：查理复用双配置嗅探分块重建 0~100 数字、闪烁频率判定充电。
- `main`：**非阻塞**主循环（闪烁采样 + 分块重建 + 周期判定），不做任何 TX。

> 说明：早期文档设想的 `Common/Common/{Battery,Charger,Dcdc,Uart}` 分层未在本简化实现中使用；如后续接入专用电量计（I2C）或需要更细分模块，可再拆分。

## 命名规范

- **文件**：下划线加小写字母，如 `main.c`
- **函数**：每个单词首字母大写加下划线，如 `Power_UART_Init()`、`Power_Charlie_ReadNumber()`
- **变量**：小写加下划线，如 `s_reported_pct`、`s_cp_map`
- **宏定义**：全大写加下划线，如 `CP_SAMPLES`、`CP_LIT_MIN`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持 WCH 官方命名

## 开发要点

1. **解码校准**：查理复用双读需落在同一扫描时隙内；若实测扫描过快/脚位不一致，按实测调 `CP_*` 参数与 `PIN_BIT()`。
2. **不抢占电源路径**：CH32V103 仅“旁路观测”数码管信号，不驱动/干扰 IP5568 的电源控制。
3. **PB3/PB4 复用**：作 GPIO 前必须关闭 SWJ（已在 `Power_SMG_Init` 以 `GPIO_Remap_SWJ_Disable` 处理）；CH32V103 无“仅关 JTAG 保留 SWD”选项，此操作会同时关闭调试口。
4. **心跳可靠性**：GET_TYPE 必须在接收中断里即时应答；万勿把应答放回阻塞式主循环轮询，否则数码管采样阻塞会拖慢应答导致 Core 高频 miss 甚至误判离线。且 TX 只能在中断内进行（单一 TX 点），避免与主循环并发写 UART。
5. **上报节流**：心跳携带当前电量/充电（~500ms 一次），Core 侧仅在变化时推 Display，不会刷屏。
6. **策略在 Core**：Power 只如实上报电量/充电，低电量降频/关背光等策略决策在 Core 侧。
