# Submodel-7 模块（副屏显示）

## 概述

本模块为辅助显示扩展方案，在主屏之外提供小型常显信息面板，用于展示设备状态、简易 LOGO 或通知提醒。本模块物理体积较大，占用两个 Submodel 插槽位置。

- **主控芯片**：CH32V103C8T6
- **屏幕规格**：2.13 寸全反屏（反射式液晶屏），分辨率 122×250
- **与核心模块接口**：模块侧 UART1，**230400/8-N-1**
- **模块 ID**：`0x40` ~ `0x42`
- **模块类型编号**：`0x05`
- **子类型编号**：`0x07`（Sub Display）
- **物理占用**：2 个 Submodel 插槽位置，占用 1 路 UART

## 硬件组成

- **CH32V103C8T6**：负责接收数据、管理显示内容、驱动屏幕刷新
- **2.13 寸全反屏**：122×250 分辨率，反射式液晶，无需背光，阳光越强光越清晰，极端低功耗
- **驱动芯片（或直驱）**：根据具体屏幕型号，可能为 UC8xxx / SSD16xx 系列驱动（SPI 接口），或 CH32V103 直接通过 SPI 发送像素数据
- **Flash（可选）**：存放预置的 LOGO 图片、图标字库

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、SPI、UART1（230400）
- 初始化全反屏驱动芯片：配置分辨率、扫描方向、刷新模式
- 加载预置资源：从 Flash 读取 LOGO 图片、图标位图到 RAM
- 向 Core 回复 `CMD_GET_TYPE`，上报子类型 `0x07`

### 运行阶段

- **内容接收**：通过 UART1 接收 Core 下发的副屏内容指令（`CMD_DISP_EXT_SUBDISP_CONTENT` / `CMD_DISP_EXT_SUBDISP_CONFIG`）
- **内容类型解析**：
  - `0x00` 关闭：清屏或进入低功耗
  - `0x01` 镜像主屏：接收主屏压缩后的状态摘要，在副屏以简化形式展示
  - `0x02` 自定义内容：显示指定的文字、图标、二维码或简单图形
  - `0x03` 状态信息：固定格式展示时间、电量、蓝牙状态、通知数量等
- **屏幕刷新**：将解析后的内容写入屏幕驱动芯片的显存，执行刷新命令
- **低功耗管理**：全反屏静态显示不耗电，CH32V103 可在刷新完成后进入睡眠模式，仅保留 UART 唤醒中断

### 中断与事件

- UART1 中断：接收 Core 的内容更新指令
- SPI / DMA 中断：屏幕数据传输完成
- 定时器中断（可选）：若需本地时钟走时，定时唤醒更新分钟 / 小时显示

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`sub_display.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`SubDisp_Update()`
- **变量**：统一小写加下划线，例如 `disp_content`、`refresh_flag`
- **文件夹**：下划线写法，仅首字母大写，例如 `SubDisplay`、`Font`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **全反屏特性**：全反屏（Reflective LCD）无背光，依靠环境光反射显示，因此黑暗环境下不可见；若需在夜间使用，可考虑在模块上加装极小面积的前光（Front Light），但会牺牲部分功耗优势。
2. **分辨率限制**：122×250 像素极低，不适合复杂 UI；建议采用固定布局分区（如顶部状态栏、中部大字体时间、底部图标行），Core 下发时只需传递字段内容，无需传输整张位图。
3. **刷新速率**：全反屏刷新通常较慢（百毫秒级），且可能有闪烁；建议在收到 Core 指令后做内容合并（如 200 ms 内连续到达的多条更新合并为一帧），减少无效刷新。
4. **与主屏的协同**：副屏是主屏的补充而非替代，建议 Core 侧维护一份“需常显信息”列表（如电量、未读通知数），在系统状态变化时自动推送到副屏；主屏熄屏后，副屏仍可保持显示。
5. **双槽位设计**：因占用 2 个 Submodel 位置，PCB 尺寸较大，布线时需注意与相邻模块的物理干涉；外壳设计预留足够空间。
6. **低功耗优势**：这是所有 Submodel 中最适合常挂运行的模块，整机休眠时可仅保留副屏显示基本信息；建议在 Power 模块的休眠策略中对此做特别考虑。
