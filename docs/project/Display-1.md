# Display-1 模块（7 寸 TFT LCD）

## 概述

本模块为液晶显示方案，负责核心底板的人机交互界面渲染，运行自研 MiniUI 框架。

- **主控芯片**：CH32H417QEU6
- **屏幕规格**：7 寸 TFT，分辨率 800×480，RGB888
- **驱动芯片**：SSD1963（FMC 8080 并口接口，内部 GRAM，PWM 背光控制）
- **与核心模块接口**：模块侧 UART1，**921600/8-N-1**
- **模块 ID**：`0x10`
- **模块类型编号**：`0x01`
- **子类型编号**：`0x01`（LCD）

## 硬件组成

- **CH32H417QEU6**：负责 UI 渲染、UART 协议解析、SSD1963 显存更新
- **SSD1963**：LCD 驱动芯片，接收 CH32H417 通过 FMC 8080 并口写入的 RGB565 数据，转换为 RGB888 存入 GRAM，以 DE 模式驱动 LCD
- **LCD 面板**：800×480 RGB888 液晶屏
- **背光电路**：SSD1963 PWM 输出控制背光亮度
- **触摸屏（如有）**：电阻或电容触摸芯片，触摸事件通过 UART 上报 Core

## 软件职责

### 初始化阶段

- 初始化 CH32H417 时钟、GPIO、FMC 8080 并口、UART1（921600）
- 初始化 SSD1963：配置分辨率、GRAM 地址、DE 模式、PWM 背光初始亮度
- 初始化 MiniUI 框架：加载字体、主题、基础控件库
- 初始化 UART 接收状态机（帧头 `0xAA`），准备接收 Core 指令
- 若支持触摸，初始化触摸芯片中断

### 运行阶段

- **UI 渲染**：根据当前页面（首页 / 应用页 / 游戏页等）执行 MiniUI 绘制，将帧缓冲通过 FMC 写入 SSD1963 GRAM
- **指令解析**：在 UART 中断中逐字节接收 Core 下发的协议帧，解析后执行对应操作（切换页面、更新状态栏、显示通知、播放控制等）
- **事件上报**：触摸事件、屏幕唤醒/休眠事件、错误异常等通过 UART 主动上报 Core
- **应用管理**：维护本地应用生命周期（启动、关闭、数据交互），本地小游戏（俄罗斯方块、2048 等）直接在 Display 模块内运行，不通过 UART 与 Core 交互

### 中断与事件

- UART1 中断：协议帧接收状态机（`WAIT_HEAD` → `FRAME_READY`）
- FMC / DMA 中断：显存批量传输完成回调
- 触摸中断（如有）：坐标采集与去抖，打包为 `CMD_DISP_TOUCH_EVT` 上报
- 背光 PWM 定时器：根据自动息屏策略调节亮度

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`ssd1963.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`SSD1963_Init()`
- **变量**：统一小写加下划线，例如 `frame_buffer`、`backlight_level`
- **文件夹**：下划线写法，仅首字母大写，例如 `MiniUI`、`SSD1963`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **波特率 921600**：UART 中断处理必须足够轻量，建议在 ISR 中仅做字节接收与状态机推进，完整帧解析放到主循环或低优先级任务中执行，避免丢字节。
2. **FMC 8080 并口**：注意读写时序与 SSD1963 数据手册匹配，GRAY 级别与 DE 模式配置正确；GRAM 地址指针自动递增特性可利用以减少总线开销。
3. **显存策略**：800×480 RGB565 全屏缓冲约 750KB，CH32H417 片内 RAM 可能不足，需评估是否使用 SSD1963 的窗口写功能配合局部刷新，或外接 SRAM/Flash 缓存资源。
4. **MiniUI 性能**：复杂页面（文件列表、图片查看）建议预渲染或分层处理，避免整屏重绘导致刷新率下降。
5. **触摸与 Core 的坐标映射**：若 V5F 通过 `CMD_DISP_VIRTUAL_TOUCH` 模拟触摸，Display 模块应统一走同一套坐标到 UI 事件的转换逻辑，避免双路径处理不一致。
6. **错误上报**：SSD1963 通信失败、触摸屏失联、UART 帧错误等应通过 `CMD_DISP_EXT_ERROR_REPORT` 及时上报，便于 Core 侧诊断。
