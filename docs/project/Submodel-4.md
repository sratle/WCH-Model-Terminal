# Submodel-4 模块（触摸圆环）

## 概述

本模块为电容触摸输入扩展方案，通过 TTP229-BSF 驱动 16 个触摸焊盘，组成 **2×2 中心方阵 + 12 位环绕圆环**，适用于音乐控制、菜单浏览、灯效调节等场景。

- **主控芯片**：CH32V103C8T6
- **与核心模块接口**：模块侧 UART1（PA9/PA10），**230400/8-N-1**
- **模块 ID**：`0x40` ~ `0x42`
- **模块类型编号**：`0x05`
- **子类型编号**：`0x04`（Touch Ring）

## 硬件组成

### 主控

- **CH32V103C8T6**：TTP229 串行读取、协议帧打包

### 触摸 — TTP229-BSF 电容触摸（16 键）

采用单片 TTP229-BSF 驱动 16 个触摸焊盘，全部 16 通道均使用。

| 引脚 | 功能 |
|------|------|
| PB15 | TTP229 SDO（数据输出） |
| PB14 | TTP229 SCL（时钟输出） |

**TTP229 配置要求**：

| 配置引脚 | 功能 | 要求 | 说明 |
|---------|------|------|------|
| TP2 (KYSEL) | 键数选择 | 串高阻电阻接 VSS → **16 键模式** | 必须为 16 键串行输出 |
| TP4 (SKMS0) | 多键模式 | 串高阻电阻接 VSS → **多键触发** | 支持多点同时触摸 |
| TP1 (SAHL) | 输出极性 | 悬空（默认低有效） | bit=0 表示触摸 |
| TP3 (SKMS1) | 多键模式 | 串高阻电阻接 VSS → 配合 TP4 实现全 16 键多键 | TP3=0, TP4=0 = 全多键触发 |

### 触摸焊盘布局

16 个触摸焊盘分为两组：

**中心方阵（2×2）**：

```
┌─────┬─────┐
│ TP9 │TP11 │    ← 上排
├─────┼─────┤
│ TP8 │TP10 │    ← 下排
└─────┴─────┘
 左列   右列
```

**环绕圆环（12 位，顺时针）**：

```
          TP13
     TP3        TP2
   TP1            TP0
  TP15            TP14
   TP7            TP6
     TP5        TP4
          TP12
```

圆环顺序（顺时针）：TP13 → TP3 → TP2 → TP1 → TP0 → TP15 → TP14 → TP7 → TP6 → TP5 → TP4 → TP12

### 引脚分配汇总

| 功能 | 引脚 | 模式 | 说明 |
|------|------|------|------|
| UART1 TX | PA9 | AF_PP | 连接 Core（230400） |

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、UART1（230400）
- 初始化 TTP229 串行接口 GPIO：PB15（SDO 输入）、PB14（SCL 推挽输出）
- 上电校准 TTP229 基线（消除 PCB 寄生电容）
- 等待 TTP229 上电稳定（约 0.5 秒）
- 向 Core 回复 `CMD_GET_TYPE`，上报子类型 `0x04`

### 运行阶段

- **触摸检测**：定时读取 TTP229 的 16-bit 串行数据，通过校准和映射转换为 16 键状态
- **事件上报**：状态变化时通过 UART1 向 Core 上报触摸事件
- **指令响应**：响应 Core 的模式设置和状态查询

### 中断与事件

- **定时器中断**：触摸扫描周期（建议 ≥ 10ms，TTP229 响应时间 32ms@16 键模式）
- **UART1 中断**：接收 Core 下发的命令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `uart_core.c`、`ttp229.h`
- **函数**：每个单词首字母大写加下划线，例如 `TTP229_Read()`、`TTP229_Calibrate()`
- **变量**：统一小写加下划线，例如 `touch_bitmap`、`raw_data`
- **文件夹**：首字母大写，例如 `TTP229`、`Uart`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **TTP229 串行时序**：MCU 等待 SDO 数据有效信号（DV，约 93μs）后发 SCL 时钟读取数据。16 键模式响应时间约 32ms，扫描周期建议 ≥ 10ms。SCL 超时 2ms 自动复位。
2. **上电校准**：TTP229 基线校准受 PCB 寄生电容影响，需在 500ms 上电稳定后捕获空闲状态作为基线，运行时用 `(~raw) & baseline` 消除寄生位。
3. **多键检测**：支持多点同时触摸（和弦/多指手势），TTP229 硬件需配置 TP4=0、TP3=0 启用全多键触发模式。
4. **圆环滑动**：12 位圆环支持滑动手势识别（顺时针/逆时针），需处理跨越 0° 点的情况，建议用环形缓冲区记录最近 N 个触摸点再做趋势判断。
5. **方阵点击**：2×2 方阵适合方向控制或功能选择，与圆环事件通过不同键 ID 区分。
