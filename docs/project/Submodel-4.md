# Submodel-4 模块（触摸圆环 / 旋钮）

## 概述

本模块为手势与旋转输入扩展方案，通过电容触摸圆环和物理旋钮提供直观的操控方式，适用于音乐控制、菜单浏览、灯效调节等场景。

- **主控芯片**：CH32V103C8T6
- **与核心模块接口**：模块侧 UART1，**115200/8-N-1**
- **模块 ID**：`0x40` ~ `0x42`
- **模块类型编号**：`0x05`
- **子类型编号**：`0x04`（Touch Ring）

## 硬件组成

- **CH32V103C8T6**：触摸芯片数据读取、编码器信号处理、协议帧打包
- **电容触摸圆环芯片**：如 TTP224 / TTP229（支持多键触摸）或专用触摸滑条/圆环芯片（如 Azoteq IQSxxx 系列）
- **物理旋转编码器**：带 A/B 相位的增量式编码器，可能集成按压按键功能
- **LED 指示（可选）**：圆环周围布置 WS2812 或普通 LED，反馈当前触摸位置或激活状态

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、外部中断、UART1（115200）
- 初始化触摸芯片：配置灵敏度、触摸键数量、输出模式（I2C / 并行 / 串行）
- 初始化旋转编码器：配置 A/B 相位引脚的外部中断或定时器输入捕获
- 校准触摸基准值（消除 PCB 寄生电容差异）
- 向 Core 回复 `CMD_GET_TYPE`，上报子类型 `0x04`

### 运行阶段

- **触摸圆环检测**：
  - 读取触摸芯片输出的激活键位或坐标值
  - 映射为圆环角度（如 0°~360°）或分段区域（如 12 点钟方向、3 点钟方向等）
  - 支持滑动手势识别：顺时针滑动、逆时针滑动、轻触点击
- **旋钮检测**：
  - 通过 A/B 相位中断判断旋转方向与步数
  - 检测旋钮按压（若有集成按键）
  - 做软件消抖与步数累积，避免抖动导致误触发
- **事件上报**：主动通过 UART1 向 Core 上报：
  - 触摸圆环事件：触摸位置、滑动手势、长按 / 双击等
  - 旋钮事件：旋转方向、旋转步数、按压状态
- **指令响应**：
  - 响应 Core 的 LED 控制指令：点亮圆环对应区域或切换灯效反馈模式
  - 响应灵敏度调节指令：调整触摸阈值或编码器倍频系数

### 中断与事件

- 触摸芯片中断（如有）：触摸状态变化
- 外部中断：编码器 A/B 相位跳变、旋钮按键按下 / 释放
- 定时器中断：长按检测、手势识别超时判断
- UART1 中断：接收 Core 的配置或灯效指令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`encoder.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Encoder_Read()`
- **变量**：统一小写加下划线，例如 `touch_pos`、`encoder_steps`
- **文件夹**：下划线写法，仅首字母大写，例如 `Touch`、`Encoder`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **触摸灵敏度**：电容触摸易受环境湿度、手指干燥程度影响，建议在模块外壳上预留灵敏度调节孔（或软件支持 Core 远程调节），出厂时做校准。
2. **编码器消抖**：机械编码器 A/B 相在旋转时会产生毛刺，软件需做状态机消抖（如记录历史 3~4 个状态，只有符合正转/反转序列时才计步），避免方向误判。
3. **手势识别**：圆环滑动手势需处理“跨越 0° 点”的情况（如从 350° 滑到 10°，应识别为顺时针 20°，而非逆时针 340°）；建议用环形缓冲区记录最近 N 个触摸点坐标再做趋势判断。
4. **双输入协同**：触摸圆环适合粗调 / 快速切换，旋钮适合精调 / 连续调节；两者事件应通过不同的事件类型码区分，便于 Core 侧路由到不同应用。
5. **LED 反馈**：若圆环带 WS2812，可在本地直接响应触摸位置点亮对应像素，无需等待 Core 指令，降低延迟；Core 只需下发模式切换指令（如“呼吸灯”、“跟随触摸”）。
