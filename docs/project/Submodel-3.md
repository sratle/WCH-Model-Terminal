# Submodel-3 模块（NFC 读卡）

## 概述

本模块为 NFC 读卡扩展方案，通过专用 NFC 模块被动接收卡片信息并上报给 Core。

- **主控芯片**：CH32V103C8T6
- **与核心模块接口**：模块侧 UART1（PA9-TX, PA10-RX），**230400/8-N-1**
- **与 NFC 模块接口**：UART2（PA2-TX, PA3-RX），**9600/8-N-1**
- **模块 ID**：`0x40` ~ `0x42`
- **模块类型编号**：`0x05`
- **子类型编号**：`0x03`（NFC）

## 硬件组成

- **CH32V103C8T6**：接收 NFC 模块数据、解析卡号、协议帧打包上报
- **NFC 读卡模块**：独立模块，通过 UART2 输出卡片数据（不可配置，仅主动上报）

### 引脚分配

| 引脚 | 功能 | 说明 |
|------|------|------|
| PA9 | UART1_TX | 与 Core 通信发送 |
| PA10 | UART1_RX | 与 Core 通信接收 |
| PA2 | UART2_TX | 与 NFC 模块通信发送（保留，当前未使用） |
| PA3 | UART2_RX | 与 NFC 模块通信接收 |

### NFC 模块通信帧格式

NFC 模块在有卡靠近时，每 **91ms** 主动输出一次 **8 字节** 数据帧：

```
+--------+----------+-------------------+----------+
| 帧头   | 路由号   | 卡号（5 字节）     | 奇偶校验 |
| 1 byte | 1 byte   | 5 bytes           | 1 byte   |
+--------+----------+-------------------+----------+
```

| 字节 | 说明 |
|------|------|
| Byte 0 | 帧头，固定 `0xAA` |
| Byte 1 | 路由号：`0x30`=第 1 路, `0x31`=第 2 路, `0x32`=第 3 路, `0x33`=第 4 路（仅 `0x30`~`0x33` 有效） |
| Byte 2~6 | 卡号，5 字节 |
| Byte 7 | 奇偶校验字节 |

> NFC 模块**不可配置**，MCU 无法向其发送任何指令。模块仅在有卡时以 9600bps 主动循环输出上述帧。

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO
- 初始化 UART1（PA9/PA10，230400bps）用于与 Core 通信
- 初始化 UART2（PA2/PA3，9600bps）用于接收 NFC 模块数据
- 向 Core 回复 `CMD_GET_TYPE`，上报子类型 `0x03`

### 运行阶段

- **NFC 数据接收**：
  - 通过 UART2 中断逐字节接收 NFC 模块输出的 8 字节帧
  - 状态机解析：帧头 `0xAA` → 路由号 → 卡号（5 字节） → 奇偶校验
  - 校验帧头与奇偶校验位，丢弃不合法帧
  - 提取路由号（`0x30`~`0x33` 映射为卡 ID 1~4）和 5 字节卡号
- **事件上报**：
  - 检测到有效卡片后，通过 UART1 向 Core 发送 `CMD_SUB_EVT_NOTIFY`（0x40），携带卡 ID 和卡号
  - 去抖处理：连续识别到 5 次相同卡号后才上报 Core，避免误报
  - 卡片离开后清除缓存，下次靠近时重新上报
- **心跳响应**：
  - 响应 Core 的 `CMD_GET_TYPE`，回复模块类型与子类型
  - 响应 Core 的 `CMD_SUB_GET_STATUS`（0x42），返回最近识别的卡信息

### 中断与事件

- UART1 中断：接收 Core 的查询指令
- UART2 中断：接收 NFC 模块输出的数据帧

## 协议帧定义

### 卡片识别上报（Module → Core）

```
CMD = 0x40 (CMD_SUB_EVT_NOTIFY)
DATA: [子命令=0x01][卡ID:1][卡号:5字节]
```

- 子命令 `0x01`：卡片识别事件
- 卡 ID：1~4（由 NFC 模块路由号 `0x30`~`0x33` 映射）
- 卡号：5 字节原始卡号

### 查询状态（Core → Module）

```
CMD = 0x42 (CMD_SUB_GET_STATUS)
DATA: [子命令=0x00]
```

**ACK 响应**：

```
CMD = 0x04 (CMD_ACK)
DATA: [卡ID:1][卡号:5字节]
```

> 若无卡，卡 ID 返回 `0x00`，卡号全 `0x00`。

## 开发要点

1. **NFC 模块帧解析**：必须严格按 8 字节帧格式解析（帧头 0xAA + 路由号 + 5 字节卡号 + 奇偶校验），使用状态机实现容错，帧头不匹配时自动重新同步。
2. **不可配置特性**：NFC 模块无法接收 MCU 的指令，仅被动输出数据。MCU 不需要向 NFC 模块发送任何内容，UART2_TX（PA2）当前保留未使用。
3. **91ms 帧间隔**：NFC 模块每 91ms 发送一次帧，MCU 需保证 UART2 中断及时处理，避免数据丢失。
4. **去抖处理**：卡片在天线边缘可能反复进入/离开检测范围，软件需做去抖（连续 5 次检测到相同卡号才上报），避免频繁重复上报。
5. **奇偶校验**：需验证每帧的校验字节，丢弃校验失败的帧，防止误报。
6. **路由号范围**：仅 `0x30`~`0x33` 有效，对应卡 ID 1~4，超出范围的帧应丢弃。
7. **被动接收模式**：整个模块逻辑简单，NFC 模块独立工作，MCU 仅做数据解析和协议转发。
