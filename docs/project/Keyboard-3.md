# Keyboard-3 模块（电子琴）

## 概述

本模块为音乐演奏输入方案，提供琴键、推子及控制按键，用于终端内置电子琴 / 合成器应用，**无 CH9329，不支持独立 USB HID 输出**。

- **主控芯片**：CH32V103C8T6
- **布局**：24 个触摸琴键（TTP229） + 3 个调音推子（ADC） + 12 个控制按钮（GPIO）
- **与核心模块接口**：模块侧 UART1（PA9/PA10），**230400/8-N-1**
- **模块 ID**：`0x20`（与 Keyboard-1/2 共用键盘模块 ID，通过子类型区分）
- **模块类型编号**：`0x04`
- **子类型编号**：`0x03`（Music Keyboard）

## 硬件组成

### 主控

- **CH32V103C8T6**：TTP229 串行读取、推子 ADC 采集、按钮 GPIO 检测、协议帧打包

### 琴键 — 双 TTP229-BSF 电容触摸（24 键）

采用两片 TTP229-BSF 驱动 24 个触摸琴键，每片 16 通道，共 32 通道，使用其中 24 个。

| 芯片 | SDO 引脚 | SCL 引脚 | 触摸引脚范围 | 琴键使用情况 |
|------|---------|---------|-------------|-------------|
| TTP229 #1 | PB15 | PB14 | TP0 ~ TP15 | 使用 TP0~TP3、TP12~TP15（8 键），TP4~TP11 未连接 |
| TTP229 #2 | PB13 | PB12 | TP16 ~ TP31 | 全部使用（16 键） |

**TTP229 配置要求（每片相同）**：

| 配置引脚 | 功能 | 要求 | 说明 |
|---------|------|------|------|
| TP2 (KYSEL) | 键数选择 | 串高阻电阻接 VSS → **16 键模式** | 必须为 16 键串行输出 |
| TP4 (SKMS0) | 多键模式 | 串高阻电阻接 VSS → **多键触发** | 音乐键盘必须支持和弦（多键同时按下） |
| TP1 (SAHL) | 输出极性 | 悬空（默认低有效）或按需配置 | 影响 SDO/SCL 极性 |
| TP3 (SKMS1) | 多键模式 | 串高阻电阻接 VSS → 配合 TP4 实现全 16 键多键 | TP3=0, TP4=0 = 全多键触发 |

> TTP229 通过二线串行接口（SDO + SCL）输出数据。MCU 等待 SDO 数据有效（DV）信号后，通过 SCL 时钟逐位读取 16 bit 按键状态（bit=1 表示触摸）。SCL 频率范围 1KHz ~ 512KHz，SCL 超时 2ms 自动复位。

### 控制按钮 — 12 个 GPIO 直连按钮

12 个机械按钮直连 MCU GPIO，两排布局（每排 6 个），**下拉到地，高电平有效（按下 = 1）**。

| 编号 | 名称 | 引脚 | 所在排 |
|-----|------|------|-------|
| 0 | KEY1 | PB9 | 第一排 |
| 1 | KEY2 | PB8 | 第一排 |
| 2 | KEY3 | PB7 | 第一排 |
| 3 | KEY4 | PB6 | 第一排 |
| 4 | KEY5 | PA4 | 第一排 |
| 5 | KEY6 | PA5 | 第一排 |
| 6 | KEY7 | PA6 | 第二排 |
| 7 | KEY8 | PA7 | 第二排 |
| 8 | KEY9 | PB0 | 第二排 |
| 9 | KEY10 | PB1 | 第二排 |
| 10 | KEY11 | PB10 | 第二排 |
| 11 | KEY12 | PB11 | 第二排 |

> GPIO 配置为 `GPIO_Mode_IPD`（内部下拉输入），读取值为 1 表示按键按下。

### 调音推子 — 3 路 ADC 电位器

3 个滑动电位器通过 ADC 采集，从左到右排列。

| 编号 | 名称 | 引脚 | ADC 通道 | 位置 |
|-----|------|------|---------|------|
| 0 | FADER_L | PA3 | ADC_CH3 | 左 |
| 1 | FADER_M | PA1 | ADC_CH1 | 中 |
| 2 | FADER_R | PA2 | ADC_CH2 | 右 |

> ADC 采样需做平滑滤波（如滑动平均），消除电位器抖动。推子值范围 0~4095（12-bit ADC），上报时转换为 0~65535（uint16）。


## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、UART1（230400）
- 初始化 TTP229 串行接口 GPIO：PB15/PB14（#1）、PB13/PB12（#2），SDO 为输入，SCL 为推挽输出
- 初始化按钮 GPIO（PB9/PB8/PB7/PB6/PA4~PA7/PB0/PB1/PB10/PB11）为内部下拉输入
- 初始化 ADC：PA1/PA2/PA3 三通道，配置 DMA 或逐通道采样
- 加载琴键映射表（TP → 键 ID）
- 等待 TTP229 上电稳定（约 0.5 秒），此期间不读取触摸数据

### 运行阶段

- **琴键扫描**：定时读取两片 TTP229 的 16-bit 串行数据，通过映射表转换为 24 键位图，检测按下 / 释放事件，支持多键同时按下（和弦）
- **推子采集**：ADC 采样三通道推子，做滑动平均滤波，值变化超过阈值时触发上报
- **按钮检测**：GPIO 读取 12 个按钮状态，做去抖处理（建议 5~10ms），检测按下 / 释放事件
- **事件上报**：事件驱动模式下，仅在状态变化时通过 UART1 向 Core 上报：
  - 琴键位图变化（3 字节，24 bit，bit=1 表示按下）
  - 推子值变化（3 × uint16 大端，6 字节，阈值约 512/65535）
  - 按钮位图变化（2 字节，12 bit，bit=1 表示按下）
- **待机模式**：默认状态，仅响应 Core 的心跳（CMD_NOP）和类型查询（CMD_GET_TYPE），不主动上报事件。收到 Core 的 `CMD_KBD_MUSIC_EVENT_CTRL`（开始输出）命令后进入事件驱动上报模式

### 中断与事件

- **定时器中断**：琴键 + 按钮扫描周期（建议 5~10ms，TTP229 响应时间 32ms@16 键模式，扫描频率不宜过高）
- **ADC 中断 / DMA 完成中断**：推子采样
- **UART1 中断**：接收 Core 下发的命令（开始/停止输出、心跳、类型查询等）

## 通信协议

### 基本行为

Keyboard-3 默认处于**待机模式**：
- 响应 `CMD_NOP`（心跳）→ 回复 `CMD_ACK`
- 响应 `CMD_GET_TYPE` → 回复 `CMD_ACK` + 模块类型信息
- **不主动上报**琴键 / 推子 / 按钮事件
- 响应 `CMD_KBD_GET_STATUS` 查询 → 回复当前状态快照

收到 Core 发送 `CMD_KBD_MUSIC_EVENT_CTRL`（state=0x01）后进入**事件驱动上报模式**：
- 每次扫描后检测状态变化（琴键/按钮/推子），仅变化时上报
- 推子变化阈值约 512/65535（~0.8%），过滤电位器抖动
- 继续响应心跳和查询命令
- 收到 `CMD_KBD_MUSIC_EVENT_CTRL`（state=0x00）后停止上报，回到待机模式

### 使用的操作码

| 操作码 | 宏名 | 方向 | 说明 |
|-------|------|------|------|
| `0x00` | `CMD_NOP` | 双向 | 心跳，Keyboard 回复 ACK |
| `0x01` | `CMD_GET_TYPE` | Core → Keyboard | 查询模块类型，Keyboard 回复 ACK + 类型数据 |
| `0x24` | `CMD_KBD_GET_STATUS` | Core → Keyboard | 查询当前输入状态快照 |
| `0x26` | `CMD_KBD_MUSIC_KEYS` | Keyboard → Core | 琴键位图上报（定时上报或查询响应） |
| `0x27` | `CMD_KBD_MUSIC_BUTTONS` | Keyboard → Core | 按钮位图上报（定时上报或查询响应） |
| `0x28` | `CMD_KBD_MUSIC_FADERS` | Keyboard → Core | 推子值上报（定时上报或查询响应） |
| `0x2A` | `CMD_KBD_MUSIC_EVENT_CTRL` | Core → Keyboard | 开始/停止事件定时上报 |

> 详细 DATA 格式见 `Protocol_Keyboard.md`。

### CMD_GET_TYPE 响应

```
DATA[0] = 0x04  (Keyboard 模块类型)
DATA[1] = 0x03  (Music Keyboard 子类型)
DATA[2] = 0x01  (硬件版本)
DATA[3] = 0x01  (固件主版本)
DATA[4] = 0x00  (固件次版本)
```

### CMD_KBD_GET_STATUS 响应

| 查询类型 (DATA[0]) | 响应内容 |
|---|---|
| `0x00` | ACK + 琴键位图（3 字节），格式同 `CMD_KBD_MUSIC_KEYS` |
| `0x01` | ACK + 按钮位图（2 字节），格式同 `CMD_KBD_MUSIC_BUTTONS` |
| `0x02` | ACK + 推子值（6 字节），格式同 `CMD_KBD_MUSIC_FADERS` |

## 开发要点

1. **无 CH9329**：本模块仅通过 UART1 与 Core 通信，不能独立接电脑使用。
2. **TTP229 串行时序**：MCU 需等待 SDO 数据有效信号（DV，约 93μs）后再发 SCL 时钟读取数据。16 键模式响应时间约 32ms，扫描周期建议 ≥ 10ms。SCL 超时 2ms 自动复位，两次读取间隔不应超过 2ms（一次完整读取内）。
3. **多键检测**：音乐键盘必须支持和弦（多键同时按下），TTP229 硬件需配置 TP4=0、TP3=0 启用全多键触发模式。
4. **TP 未使用通道**：TTP229 #1 的 TP4~TP11 未连接琴键，固件读取后应屏蔽这些位（置 0），避免误报。TTP229 #2 全部 16 通道均用于琴键。
5. **推子滤波**：电位器 ADC 原始值存在噪声，建议采用滑动平均滤波（窗口 8~16 次采样），并设置变化阈值（如 ±16），避免微小抖动触发上报。
6. **按钮去抖**：机械按钮存在弹跳，建议软件去抖 5~10ms（连续 N 次采样一致才确认状态变化）。
7. **待机/上报模式切换**：Keyboard-3 上电后默认待机，只有 Core 发送开始命令后才启动事件驱动上报。这避免了非音乐场景下的无效数据流量。上报采用变化触发而非定时轮询，仅在琴键/按钮/推子状态发生变化时才向 Core 发送事件帧。
8. **与 Core 的音频闭环**：Keyboard-3 只负责输入采集，音频合成与播放由 Core 侧的 CS43131 完成。音符编号建议直接使用 MIDI note number（键 ID + 59 = MIDI note），降低 Core 侧解析复杂度。
9. **控制按钮鼓点映射（Core 侧）**：12 个控制按钮 KEY1~KEY12 在 Core 侧顺序映射到 `\SOUND\DRUM-01.WAV ~ DRUM-12.WAV`（一击即发的打击乐/鼓点音色）。Core 收到按钮位图（`CMD_KBD_MUSIC_BUTTONS` 0x27）后做上升沿检测，仅在按钮由“松”变“按”时触发一次对应鼓点，松开不主动停止，让 WAV 自然衰减结束。鼓点与琴键共用 CS43131 的 ch0/ch1 双通道轮转：每个新音色（无论琴键还是鼓点）都分配到与上一次不同的通道（严格交替），使上一个仍在延音的音色留在原通道继续、不被打断，保证播放流畅。
