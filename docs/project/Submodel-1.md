# Submodel-1 模块（指纹识别）

## 概述

本模块为生物识别扩展方案，用于用户身份验证、解锁终端及支付授权等场景。

- **主控芯片**：CH32V103C8T6
- **与核心模块接口**：UART1（PA9-TX，PA10-RX），**230400/8-N-1**
- **与指纹传感器接口**：UART2（PA2-TX，PA3-RX），**57600/8-N-1**（指纹模块默认波特率）
- **手指检测**：PA4 连接指纹传感器 TOUCHOUT 引脚，下降沿触发中断
- **模块 ID**：`0x40` ~ `0x42`（根据插入的物理槽位分配）
- **模块类型编号**：`0x05`
- **子类型编号**：`0x01`（Fingerprint）

## 硬件组成

- **CH32V103C8T6**：指纹模块控制、协议帧转发、模板管理
- **指纹传感器**：采用 Syno 协议（包头 `0xEF01`）的光学/电容式指纹采集模组（如 ZW101、AS608 等型号），通过 UART2 与主控通信，内置 DSP 和 Flash，支持自动注册、自动验证、灯效控制等功能
- **TOUCHOUT 引脚**：传感器手指检测输出，连接至 PA4，手指按下时输出低电平，用于中断唤醒和轮询触发

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、UART1（230400）、UART2（57600）
- 初始化 PA4 为下拉输入，配置 EXTI 中断（下降沿触发）
- 初始化指纹传感器：发送握手指令（`PS_HandShake` / `0x35`），验证通信正常
- 读取指纹库有效模板数量（`PS_ValidTempleteNum` / `0x1D`）
- 向 Core 回复 `CMD_GET_TYPE`，上报子类型 `0x01`

### 运行阶段

- **自动验证模式（默认）**：
  - PA4 中断触发或轮询检测到手指按下后，发送 `PS_AutoIdentify`（`0x32`）启动自动验证
  - 验证成功：向 Core 发送 `CMD_SUB_EVT_NOTIFY SUB=0x01`，携带指纹 ID 和名字
  - 验证失败：向 Core 发送 `CMD_SUB_EVT_NOTIFY SUB=0x02`，携带错误码
  - 验证完成后传感器自动进入低功耗待机
- **注册流程**（由 Core 发起）：
  - 收到 `CMD_SUB_SET_MODE SUB=0x01` 后，发送 `PS_AutoEnroll`（`0x31`）启动自动注册
  - 注册成功：向 Core 发送 `CMD_SUB_ACTION_RESULT SUB=0x01`，携带新指纹 ID 和名字
  - 注册失败：向 Core 发送 `CMD_SUB_ACTION_RESULT SUB=0x02`，携带错误码
  - 收到 `CMD_SUB_SET_MODE SUB=0x02` 后，发送 `PS_Cancel`（`0x30`）取消注册
- **指令响应**：
  - 响应删除指纹指令：发送 `PS_DeletChar`（`0x0C`）
  - 响应清空指纹指令：发送 `PS_Empty`（`0x0D`）
  - 响应查询指令：发送 `PS_ReadIndexTable`（`0x1F`）读取索引表
  - 响应灯效控制指令：发送 `PS_ControlBLN`（`0x3C`）控制呼吸灯
- **安全策略**：连续识别失败多次后，暂时锁定并上报 Core

### 中断与事件

- **PA4 EXTI 中断**：手指按压检测（TOUCHOUT 下降沿），唤醒主控进入验证流程
- **UART1 中断**：接收 Core 的注册 / 删除 / 查询 / 配置指令
- **UART2 中断/轮询**：接收指纹传感器的应答数据（Syno 协议解析）

## 与指纹传感器的通信

指纹传感器采用 Syno 协议，数据包格式为：

```
包头(2B:0xEF01) + 设备地址(4B:默认0xFFFFFFFF) + 包标识(1B) + 包长度(2B) + 指令/数据(NB) + 校验和(2B)
```

### 使用的传感器指令

| 传感器指令 | 指令码 | 用途 | 说明 |
|-----------|--------|------|------|
| PS_AutoEnroll | 0x31 | 自动注册 | 一站式完成指纹注册全流程，参数：ID号、录入次数、参数(2B) |
| PS_AutoIdentify | 0x32 | 自动验证 | 一站式完成指纹验证全流程，参数：分数等级、ID号、参数(2B) |
| PS_Cancel | 0x30 | 取消操作 | 取消当前自动注册/验证流程 |
| PS_DeletChar | 0x0C | 删除模板 | 删除指定ID开始的N个模板 |
| PS_Empty | 0x0D | 清空指纹库 | 删除所有指纹模板 |
| PS_ReadIndexTable | 0x1F | 读索引表 | 读取录入模板的索引表（用于查询指纹列表） |
| PS_ValidTempleteNum | 0x1D | 读有效模板数 | 读取指纹库中有效模板数量 |
| PS_ControlBLN | 0x3C | 呼吸灯控制 | 控制呼吸灯状态和效果 |
| PS_HandShake | 0x35 | 握手 | 检测模组是否正常工作 |
| PS_Sleep | 0x33 | 休眠 | 设置传感器进入休眠模式 |
| PS_ReadSysPara | 0x0F | 读系统参数 | 读取模组基本通信和配置参数 |
| PS_WriteReg | 0x0E | 写系统寄存器 | 写模组寄存器（用于设置安全等级等） |

### Syno 协议确认码（常用）

| 确认码 | 含义 |
|--------|------|
| 0x00 | 指令执行成功 |
| 0x02 | 传感器上没有手指 |
| 0x08 | 指纹不匹配 |
| 0x09 | 没搜索到指纹 |
| 0x1E | 自动注册失败 |
| 0x1F | 指纹库满 |
| 0x27 | 指纹已存在 |
| 0x58 | 自动操作终止成功 |

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`fingerprint.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Fp_Enroll()`
- **变量**：统一小写加下划线，例如 `fp_id`、`match_score`
- **文件夹**：下划线写法，仅首字母大写，例如 `Fingerprint`、`Flash`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **指纹传感器选型**：采用支持 Syno 协议的指纹模组（如 ZW101、AS608），内置 DSP 和 Flash，主控只需发送指令和读取结果，无需在 CH32V103 上运行指纹识别算法，大幅降低开发难度。
2. **双 UART 架构**：UART1（230400）负责与 Core 通信（统一协议帧），UART2（57600）负责与指纹传感器通信（Syno 协议）。两个协议栈独立运行，模块作为协议转换桥梁。
3. **TOUCHOUT 中断**：PA4 连接传感器 TOUCHOUT 引脚，手指按下时产生下降沿中断。此引脚在传感器休眠后仍然有效，可用于唤醒模块进入验证流程。初始化时需注意：Syno 协议文档指出模组上电后中断脚不应作为中断使用（可能产生误触发），建议上电延时后（约 80ms）再使能中断，或采用轮询 + 中断混合策略。
4. **自动注册/验证**：优先使用 `PS_AutoEnroll`（`0x31`）和 `PS_AutoIdentify`（`0x32`）模块指令，传感器内部自动完成采图→生成特征→合并→存储/搜索的全流程，主控无需手动分步控制。
5. **灯效控制**：传感器支持 `PS_ControlBLN`（`0x3C`）控制板载 LED，可用于注册/验证过程中的视觉反馈。若传感器处于休眠状态，需先唤醒再发送灯效指令，或缓存命令待唤醒后发送。
6. **模板管理**：传感器自带 Flash 存储（通常支持 100~300 枚指纹），通过 `PS_ReadIndexTable`（`0x1F`）可读取索引表，用于构建指纹列表回复 Core。
7. **隐私安全**：指纹模板数据不应以明文形式通过 UART 传输；若 Core 请求读取原始模板（如备份），建议做简单异或或加密处理。
8. **与 Core 的协同**：指纹注册流程通常需要 UI 引导（提示用户按多次手指），因此注册指令一般由 Core 发起，模块执行采集与存储，完成后上报结果。注册过程中 Core 可通过灯效命令控制 LED 给用户视觉反馈。
