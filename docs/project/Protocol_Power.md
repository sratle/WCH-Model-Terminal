# Power 模块通信协议（简化版）

## 1. 模块概述

- **模块类型编号**：`0x03`
- **模块 ID**：`0x30`（预留范围 `0x30~0x31`）

Power 模块基于 IP5568 电源管理芯片实现供电/充放电，**主控 CH32V103C8T6 不直接参与电源路径控制**，仅：

1. 回复 Core 的 `CMD_GET_TYPE` 心跳，以证明 Power 模块已接入；
2. 解析 IP5568 驱动数码管（电量指示）的信号，换算成 `0~100` 的电量百分比，并在**电量变化时**上报给 Core。

> **能力边界**：受硬件限制，Power 模块**只能上报当前电量**。不提供电压/电流/温度/功率、充电策略、输出策略、告警阈值等能力（旧协议中的相关操作码已废弃）。

| 特性 | 参数 |
|------|------|
| 主控 | CH32V103C8T6 |
| 电源管理芯片 | IP5568（PD 快充 / 无线充 / 5V 稳压输出 / 数码管电量指示） |
| 电池容量 | 6000 mAh |
| 电量来源 | 解析 IP5568→数码管（SMG1~SMG5）驱动信号 |

### 1.1 帧格式速查

Power 模块侧使用 **USART1**、Core 侧使用 **UART5** 互连，物理层参数：

| 参数 | 值 |
|------|-----|
| 波特率 | 230400 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 流控 | 无 |

**帧结构**：

```
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
|  HEAD  |  SRC   |  DST   |  LEN   |  CMD   |  DATA[0..N] | TAIL0  | TAIL1  | TAIL2  | TAIL3  |
| 1 byte | 1 byte | 1 byte | 1 byte | 1 byte |   N bytes   | 1 byte | 1 byte | 1 byte | 1 byte |
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
```

- **HEAD**：固定 `0xAA`。
- **LEN**：`CMD + DATA` 的总字节数（`N = LEN - 1`）。
- **TAIL**：固定序列 `A5 5A FC FD`，任一字节不匹配则丢弃整帧。

**常用模块 ID**：

| ID | 模块 |
|----|------|
| `0x00` | Core（主控） |
| `0x30` | Power（本模块） |

> 完整协议规范（接收状态机、通用操作码 `0x00~0x0F`、NACK 错误码）见 `PROJECT.md`。

## 2. 操作码定义

Power 模块专用操作码范围 `0x31 ~ 0x3F`，简化后**仅保留电量上报**。

| 操作码 | 宏名 | 方向 | 说明 | DATA 格式 |
| --- | --- | --- | --- | --- |
| `0x36` | `CMD_PWR_STATUS_REPORT` | Power -> Core | 电量/充电状态变化上报（事件类，无需响应） | `[电量:1][充电:1]` |
| `0x31~0x35, 0x37~0x3F` | — | — | **已废弃/预留**（简化协议移除充电/电压/电流/温度/告警等） | — |

> Core 的心跳通过通用操作码 `CMD_GET_TYPE`(0x01) 下发，Power 以 `CMD_ACK`(0x04) 回复身份（见 §4）。

---

## 3. 交互规则

| 语义 | 命令 | 接收方行为 |
|------|------|-----------|
| **心跳** | Core → Power `CMD_GET_TYPE`(0x01) | Power 回复 `CMD_ACK` + 身份数据（证明在线） |
| **电量上报** | Power → Core `CMD_PWR_STATUS_REPORT`(0x36) | Core **不回复**；仅在电量数值变化时上报 |

- **无查询/设置类命令**：Core 不再向 Power 发送 GET_STATUS / SET_* 等；Power 收到未知操作码可静默丢弃。
- **无自动重传**：依赖物理层可靠性。

## 4. 数据格式

### 4.1 电量/充电上报（CMD 0x36，Power → Core）

```
DATA[0]: 电量百分比（1 字节，0~100）
DATA[1]: 充电状态（1 字节，0=放电, 1=充电中）
```

> 总 DATA 长度：2 字节，`LEN = 3`。仅在电量或充电状态相较上次发生变化时发送。
> 兼容：若只发 1 字节（旧固件），Core 视充电状态为 0。

**充电状态判定依据（IP5568 188 型数码管显示规律）**：

| 显示 | 含义 | 上报 |
|------|------|------|
| 整屏 0.5Hz 闪烁 | 充电中（0~99% 未充满） | 充电=1 |
| 常亮 100% | 已充满 | 充电=0，电量=100 |
| 整屏 1Hz 闪烁 | 放电且电量 <5%（危急） | 充电=0，电量≈5 |
| 常亮（非满） | 放电（5~100%） | 充电=0 |

> Power 侧通过检测数码管整屏亮灭的**闪烁频率**推断充电状态（0.5Hz=充电，1Hz=低电放电，常亮=放电/已满）。

**示例**：电量 60%、充电中
```
[AA][30][00][03][36][3C][01][A5][5A][FC][FD]
 HEAD SRC DST LEN CMD  60  充电 ----TAIL----
```

### 4.2 CMD_GET_TYPE 响应（Power → Core）

Power 收到 `CMD_GET_TYPE`(0x01) 后以 `CMD_ACK`(0x04) 回复：

| 字节 | 字段 | 值/说明 |
|------|------|---------|
| DATA[0] | 模块类型编号 | `0x03` |
| DATA[1] | 模块子类型编号 | `0x01`（`MODULE_SUBTYPE_POWER_STANDARD`） |
| DATA[2] | 硬件版本号 | 可选，无则 `0x00` |
| DATA[3] | 固件主版本号 | 可选 |
| DATA[4] | 固件次版本号 | 可选 |

**示例**：Core 查询类型 / Power 回复
```
Core:  [AA][00][30][01][01][A5][5A][FC][FD]                     GET_TYPE
Power: [AA][30][00][06][04][03][01][01][01][00][A5][5A][FC][FD] ACK: type=03 sub=01 hw=01 fw=1.0
```

**Power 子类型定义**（`protocol.h` → `MODULE_SUBTYPE_POWER_*`）：

| 子类型编号 | 宏名 | 说明 |
|------------|------|------|
| `0x00` | `MODULE_SUBTYPE_POWER_RESERVED` | 保留 |
| `0x01` | `MODULE_SUBTYPE_POWER_STANDARD` | 标准供电模块（当前唯一实现） |
