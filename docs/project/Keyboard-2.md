# Keyboard-2 模块（游戏手柄）

## 概述

本模块为游戏操控输入方案，采用方向键 + 摇杆 + 功能键布局，既服务于终端游戏场景，也可作为独立 USB HID 游戏控制器连接电脑。

- **主控芯片**：CH32V103C8T6
- **布局**：方向键（上/下/左/右）+ 摇杆 + 4 个自定义按键
- **与核心模块接口**：模块侧 UART1，**230400/8-N-1**
- **模块 ID**：`0x20`（与 Keyboard-1 共用键盘模块 ID，通过子类型区分）
- **模块类型编号**：`0x04`
- **子类型编号**：`0x02`（Game Keyboard）

## 硬件组成

- **CH32V103C8T6**：按键扫描、摇杆 ADC 采集、协议帧打包、CH9329 通信
- **方向键**：4 向机械按键或导电胶按键
- **摇杆**：双轴模拟摇杆（2 路 ADC 输入）+ 摇杆按下按键
- **功能键**：4 个自定义按键（A/B/X/Y 或功能定义）
- **CH9329**：串口转 HID 芯片，配置为游戏控制器 / 摇杆模式
- **USB-A 接口**：Device 口，输出 HID 信号

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、ADC、UART1（230400）、UART2（接 CH9329）
- 初始化方向键与功能键 GPIO，配置内部上拉
- 初始化 ADC：配置双轴摇杆的 X/Y 通道采样
- 初始化 CH9329：设置为摇杆 / 游戏手柄 HID 报告模式
- 加载按键映射表

### 运行阶段

- **按键扫描**：定时扫描方向键与功能键，检测按下 / 释放 / 组合事件
- **摇杆采集**：定时触发 ADC 采样，读取摇杆 X/Y 轴位置及按下状态，进行死区处理与归一化
- **双通道输出**：
  - 通过 **UART1** 向 Core 上报游戏控制事件（可复用 `CMD_KBD_KEY_DOWN` / `CMD_KBD_KEY_UP`，或在 DATA 中标识设备类型为游戏手柄）
  - 通过 **UART2 → CH9329 → USB-A** 输出标准 HID 摇杆 / 游戏手柄报告
- **背光 / 震动（如有）**：接收 Core 指令控制 RGB 指示灯或震动马达反馈

### 中断与事件

- 定时器中断：按键扫描与摇杆 ADC 采样周期（建议 5~10 ms）
- ADC 中断：采样完成回调，更新摇杆坐标缓存
- UART1 中断：接收 Core 下发的配置或灯效指令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`gamepad.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Joystick_Read()`
- **变量**：统一小写加下划线，例如 `joy_x`、`btn_a_pressed`
- **文件夹**：下划线写法，仅首字母大写，例如 `Gamepad`、`CH9329`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **摇杆死区处理**：模拟摇杆存在机械中位漂移，建议在软件中设置 5%~10% 的死区，避免无操作时持续上报微小偏移。
2. **CH9329 模式配置**：需确认 CH9329 是否原生支持游戏手柄报告描述符；若不支持，可能需要将其配置为自定义 HID 设备，或由 Core 侧通过蓝牙转发摇杆数据。
3. **ADC 采样稳定性**：摇杆电位器供电建议使用 LDO 稳定电压，避免电源波动导致 ADC 值跳变；采样周期不宜过短（避免占用过多 CPU），也不宜过长（避免操控延迟）。
4. **与 Keyboard-1 的协议兼容**：两者共用 `0x20` 模块 ID，Core 通过 `CMD_GET_TYPE` 子类型（`0x01` Main / `0x02` Game）区分；上报键码时建议在 DATA 首字节增加设备子类型标识，便于 Core 侧路由到对应应用（文本编辑器 vs 游戏引擎）。
5. **接电脑即插即用**：通过 CH9329 独立输出 HID，意味着即使不插入移动终端，单独把手柄模块接电脑也应能正常工作，这是该模块的重要卖点。
