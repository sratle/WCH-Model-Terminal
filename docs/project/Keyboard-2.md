# Keyboard-2 模块（游戏手柄）

## 概述

本模块为游戏操控输入方案，提供丰富的游戏控制输入能力，既服务于终端游戏场景，也可作为独立 USB HID 设备连接电脑。

- **主控芯片**：CH32V103C8T6
- **输入设备**：6 个按钮 + 3 个钮子开关 + 2 个摇杆 + 2 个旋转编码器
- **与核心模块接口**：模块侧 UART1（PA9-TX / PA10-RX），**230400/8-N-1**
- **CH9329 接口**：UART2（PA2-TX / PA3-RX），**115200/8-N-1**
- **模块 ID**：`0x20`（与 Keyboard-1 共用键盘模块 ID，通过子类型区分）
- **模块类型编号**：`0x04`
- **子类型编号**：`0x02`（Game Keyboard）

## 硬件组成

- **CH32V103C8T6**：GPIO 按键扫描、ADC 摇杆采集、编码器计数、协议帧打包、CH9329 通信
- **6 个按钮**（BUTTON1~BUTTON6）：机械按键，GPIO 直连，内部上拉
  - PB10 → BUT1，PB11 → BUT2，PB12 → BUT3
  - PB13 → BUT4，PB14 → BUT5，PB15 → BUT6
- **3 个钮子开关**（SW1~SW3）：每个开关有 UP/DOWN 两路 GPIO，内部上拉
  - PB0 → SW1_UP，PB1 → SW1_DOWN
  - PA13 → SW2_UP，PA14 → SW2_DOWN
  - PB4 → SW3_UP，PB5 → SW3_DOWN
- **2 个摇杆**（ROC1~ROC2）：双轴模拟摇杆，4 路 ADC 输入
  - PA4 → ADC_IN4 → ROC1_X
  - PA5 → ADC_IN5 → ROC1_Y
  - PA6 → ADC_IN6 → ROC2_X
  - PA7 → ADC_IN7 → ROC2_Y
- **2 个旋转编码器**（EC1~EC2）：增量式编码器，A/B 正交信号 + D 按下信号
  - PB6 → EC1_A，PB7 → EC1_B，PA11 → EC1_D（按下）
  - PB8 → EC2_A，PB9 → EC2_B，PA12 → EC2_D（按下）
- **CH9329**：串口转 HID 芯片，UART2 连接，将输入转换为 USB HID 报告
- **USB-A 接口**：Device 口，输出 HID 信号，可接 PC 等主机

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO
- 初始化 UART1（230400, 8-N-1）：PA9-TX / PA10-RX，连接 Core
- 初始化 UART2（115200, 8-N-1）：PA2-TX / PA3-RX，连接 CH9329
- 初始化 GPIO：6 个按钮 + 3 个钮子开关（6 路）+ 2 个编码器 D 信号，配置内部上拉
- 初始化 ADC：配置 4 通道（IN4~IN7）用于摇杆 X/Y 采样
- 初始化编码器：A/B 正交解码，可用外部中断或定时器编码器模式
- 初始化 CH9329：设置为键盘模式（协议模式 0）
- 加载按键映射表

### 运行阶段

- **按钮扫描**：定时扫描 6 个按钮，检测按下 / 释放事件，软件消抖（5~10ms）
- **钮子开关扫描**：定时扫描 3 个开关的 UP/DOWN 状态，检测拨动事件
- **摇杆采集**：定时触发 ADC 采样（建议 10~20ms），读取 4 通道 X/Y 值，进行死区处理（5%~10%）与归一化至 int8（-128~127）
- **编码器解码**：检测 A/B 正交信号变化，累计旋转增量（顺时针/逆时针），支持按下检测
- **双通道输出**：
  - 通过 **UART1** 向 Core 上报游戏控制事件（`CMD_KBD_GAME_INPUT`）
  - 通过 **UART2 → CH9329 → USB-A** 输出标准 HID 报告（键盘/鼠标模式），实现接电脑即插即用

### 中断与事件

- 定时器中断：按钮/开关扫描与摇杆 ADC 采样周期（建议 5~10ms）
- ADC 中断（可选）：采样完成回调，更新摇杆坐标缓存
- 外部中断（可选）：编码器 A/B 边沿触发，实时解码旋转方向
- UART1 中断：接收 Core 下发的配置或指令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`gamepad.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Joystick_Read()`
- **变量**：统一小写加下划线，例如 `joy_x`、`btn_a_pressed`
- **文件夹**：下划线写法，仅首字母大写，例如 `Gamepad`、`CH9329`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **摇杆死区处理**：模拟摇杆存在机械中位漂移，建议在软件中设置 5%~10% 的死区，避免无操作时持续上报微小偏移。ADC 12-bit（0~4095）映射至 int8（-128~127）。
2. **编码器解码**：旋转编码器 A/B 正交信号需做边沿检测与方向判断。可使用外部中断（EXTI）或定时器编码器模式。编码器增量值在主循环中读取后清零。
3. **钮子开关处理**：每个开关有 UP/DOWN 两路 GPIO，低电平有效（内部上拉），同一时刻 UP 和 DOWN 不应同时为低。可编码为三态：上拨（UP=0, DOWN=1）、下拨（UP=1, DOWN=0）、中间（UP=1, DOWN=1）。
4. **CH9329 模式**：CH9329 原生支持键盘和鼠标 HID 报告，不支持游戏手柄报告描述符。游戏手柄输入需映射为键盘按键或鼠标操作输出给 CH9329，详见 HID 映射方案。
5. **双通道输出一致性**：同一输入事件需同时向 Core（自定义协议 `CMD_KBD_GAME_INPUT`）和 CH9329（HID 报告）输出，两者映射表独立维护。接电脑时无需依赖 Core 即可工作。
6. **与 Keyboard-1 的协议兼容**：两者共用 `0x20` 模块 ID，Core 通过 `CMD_GET_TYPE` 子类型（`0x01` Main / `0x02` Game）区分。
7. **ADC 采样稳定性**：摇杆电位器供电建议使用 LDO 稳定电压，避免电源波动导致 ADC 值跳变。采样周期建议 10~20ms。
8. **接电脑即插即用**：通过 CH9329 独立输出 HID，意味着即使不插入移动终端，单独把游戏手柄模块接电脑也应能正常工作，这是该模块的重要卖点。
