# Submodel-6 模块（激光测距）

## 概述

本模块为激光测距扩展方案，采用 VL53L0X-V2 ToF 激光测距传感器，通过飞行时间（Time-of-Flight）技术测量前方障碍物距离，可用于手势感应、物体接近检测或简易测距应用。

- **主控芯片**：CH32V103C8T6
- **与核心模块接口**：模块侧 UART1（PA9-TX, PA10-RX），**230400/8-N-1**
- **模块 ID**：`0x40` ~ `0x42`（由 Core 通过 GET_TYPE 动态分配）
- **模块类型编号**：`0x05`
- **子类型编号**：`0x06`（Infrared）

## 硬件组成

- **CH32V103C8T6**：传感器数据采集、滤波处理、协议帧打包
- **VL53L0X-V2 激光测距传感器**：ToF 飞行时间测距，I2C 接口，默认模式量程 30~1200 mm，精度 ±3%
  - **SCL**：PB15（I2C 时钟）
  - **SDA**：PB14（I2C 数据）
  - **GPIO1**：PB13（传感器中断输出，数据就绪标志）
  - **XSHUT**：PB12（传感器硬件复位/关断控制）

### 引脚分配

| 引脚 | 功能 | 说明 |
|------|------|------|
| PA9 | UART1_TX | 与 Core 通信发送 |
| PA10 | UART1_RX | 与 Core 通信接收 |
| PB15 | I2C_SCL | VL53L0X I2C 时钟 |
| PB14 | I2C_SDA | VL53L0X I2C 数据 |
| PB13 | GPIO1 | VL53L0X 数据就绪中断 |
| PB12 | XSHUT | VL53L0X 复位/关断控制 |

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、I2C（PB14/PB15）、UART1（PA9/PA10, 230400）
- 通过 XSHUT（PB12）复位 VL53L0X，等待传感器就绪
- 初始化 VL53L0X 传感器：配置默认测距模式、I2C 地址（0x29）
- 模块 ID 初始为 `0x00`（未知），等待 Core 发送 GET_TYPE 时从 DST 字段学习

### 运行阶段

- **槽位学习**：
  - 收到 Core 的 GET_TYPE 命令时，从帧的 DST 字段（0x40/0x41/0x42）学习自身模块 ID
  - 后续所有响应帧的 SRC 字段使用学习到的模块 ID
- **测距控制**：
  - 收到 SET_MODE SUB=0x01（开始测距）后，启动 VL53L0X 连续测距模式
  - 收到 SET_MODE SUB=0x02（停止测距）后，停止 VL53L0X 测距，传感器进入待机
- **GET_TYPE 响应与测距数据上报**：
  - 始终回复 GET_TYPE 的 ACK（模块类型 0x05，子类型 0x06）
  - 若测距处于激活状态，在 GET_TYPE ACK 之后紧接着发送测距结果帧
  - 若测距未激活，仅回复 GET_TYPE ACK，不发送测距数据
- **滤波处理**：对原始距离值做滑动平均或中值滤波，消除单次测量跳变

### 中断与事件

- EXTI 中断：PB13（GPIO1）下降沿触发，VL53L0X 数据就绪
- UART1 中断：接收 Core 的查询或配置指令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`vl53l0x.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`VL53L0X_Read()`
- **变量**：统一小写加下划线，例如 `distance_mm`、`i2c_buf`
- **文件夹**：下划线写法，仅首字母大写，例如 `VL53L0X`、`Filter`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **VL53L0X I2C 地址**：默认地址 0x29（7-bit），若系统中有多个 VL53L0X，需通过 XSHUT 逐个唤醒并重新分配地址。本模块仅使用一个 VL53L0X，无需地址分配。
2. **GPIO1 中断模式**：VL53L0X 的 GPIO1 引脚在测距数据就绪时拉低，可用于触发外部中断，避免 I2C 轮询。建议配置 PB13 为下降沿触发的外部中断。
3. **XSHUT 控制时序**：XSHUT 拉低至少 100μs 后释放，传感器执行硬件复位；复位后需等待约 1.2ms 再进行 I2C 通信。初始化时应先通过 XSHUT 复位传感器，确保已知状态。
4. **测距模式选择**：VL53L0X 支持多种测距模式（默认、高精度、长距离、高速），默认模式平衡了精度和量程（30~1200mm，±3%），适合大多数场景。高精度模式量程缩短但精度提升至 ±1mm。
5. **滤波策略**：VL53L0X 单次测量可能因目标表面反射率不同而波动，建议使用滑动平均（窗口 4~8）或中值滤波消除跳变。
6. **与 Core 的协同**：距离数据通过 GET_TYPE 轮询机制上报，Core 控制 GET_TYPE 的查询频率即控制了距离数据的更新频率。策略逻辑在 Core 侧实现。
