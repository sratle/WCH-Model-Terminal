# Submodel-6 模块（红外测距）

## 概述

本模块为距离检测扩展方案，通过红外传感器测量前方障碍物距离，可用于手势感应、物体接近检测或简易测距应用。

- **主控芯片**：CH32V103C8T6
- **与核心模块接口**：模块侧 UART1，**230400/8-N-1**
- **模块 ID**：`0x40` ~ `0x42`
- **模块类型编号**：`0x05`
- **子类型编号**：`0x06`（Infrared）

## 硬件组成

- **CH32V103C8T6**：传感器数据采集、滤波处理、协议帧打包
- **红外测距传感器**：如夏普 GP2Y0A21YK0F（模拟输出，10~80 cm）或 GP2Y0A02YK0F（20~150 cm）；亦可采用数字接口的 ToF 传感器（如 VL53L0X / VL53L1X，I2C 接口）作为更高精度方案
- **ADC 或 I2C**：根据传感器选型确定接口类型
- **透镜 / 遮光罩（可选）**：减少环境红外光干扰，限定检测角度

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、ADC（模拟传感器）或 I2C（数字传感器）、UART1（230400）
- 初始化红外传感器：配置测量模式、采样率、量程
- 建立距离 lookup 表（针对模拟传感器，将 ADC 电压映射为距离）
- 向 Core 回复 `CMD_GET_TYPE`，上报子类型 `0x06`

### 运行阶段

- **距离采集**：
  - 模拟传感器：定时触发 ADC 采样，读取电压值，查表或公式转换为距离（cm）
  - 数字传感器：通过 I2C 读取测距结果，部分传感器需等待测量完成标志
- **滤波处理**：对原始距离值做滑动平均或中值滤波，消除单次测量跳变
- **事件上报**：主动通过 UART1 向 Core 上报：
  - 连续距离数据流（如每 100 ms 上报一次）
  - 接近 / 远离事件（如距离从 >50 cm 进入 <10 cm 范围）
- **指令响应**：
  - 响应 Core 的查询指令：返回当前距离值
  - 响应配置指令：修改采样周期、滤波窗口大小、检测阈值

### 中断与事件

- ADC / I2C 中断：传感器数据采集完成
- 定时器中断：周期性采样节拍
- UART1 中断：接收 Core 的查询或配置指令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`range.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Range_Read()`
- **变量**：统一小写加下划线，例如 `distance_mm`、`adc_raw`
- **文件夹**：下划线写法，仅首字母大写，例如 `Range`、`Filter`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **传感器选型权衡**：夏普 GP2 系列为传统红外三角测距，成本低但精度一般（±1~2 cm），且对深色/透明物体误差大；VL53L0X 为 ToF 激光测距，精度高（±1 mm）、体积小，但价格稍高且 I2C 地址固定（多传感器时需外接 I2C 切换器）。建议根据目标精度与成本选型。
2. **模拟传感器标定**：GP2Y0A21YK0F 的输出电压-距离曲线非线性，且不同批次有差异；建议每个模块在出厂时做多点标定（如 10 cm、20 cm、30 cm、50 cm、80 cm），将标定数据写入 Flash，运行时插值计算。
3. **环境光干扰**：红外传感器在强光下（尤其含红外成分的阳光、白炽灯）可能产生误判；模块外壳建议做红外滤光或物理遮光槽，软件层面也可在异常跳变时标记数据可信度。
4. **手势识别（进阶）**：若采样频率足够高（>20 Hz），可通过距离变化趋势识别简单手势（如靠近、远离、悬停）；Submodel-6 可本地做基础手势判断后上报事件，减少 Core 侧负担。
5. **与 Core 的协同**：距离数据通常由 Core 路由到 Display 展示（如数字读数、波形图）或触发系统级动作（如接近亮屏、远离息屏）；Submodel-6 只需按配置周期上报距离值，策略逻辑在 Core。
