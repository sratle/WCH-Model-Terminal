# Submodels 模块通信协议

## 1. 模块概述

- **模块类型编号**：`0x05`
- **模块 ID**：`0x40` ~ `0x42`（对应 UART6/7/8 三个配件槽）

Submodels 为可热插拔的配件模块，通过 UART 与 Core（V3F）通信。整机最多同时插入 **2 ~ 3** 个 Submodel（Submodel-7 副屏占用两个物理位置）。Core 通过 `CMD_GET_TYPE` 识别每个槽位实际插入的子类型。

| 子类型 | 名称 | 功能 |
|--------|------|------|
| `0x01` | Fingerprint | 指纹识别 |
| `0x02` | Health | 健康监测（心跳/血氧/体温） |
| `0x03` | NFC | NFC 读卡 |
| `0x04` | Touch Ring | 触摸圆环 + 旋钮 |
| `0x05` | RGB | RGB 灯效控制 |
| `0x06` | Infrared | 红外测距 |
| `0x07` | Sub-Display | 副屏（反射屏，黑白低分辨率） |

### 1.1 帧格式速查

Submodels 通过 **UART6/7/8** 与 Core（V3F）通信，物理层参数：

| 参数 | 值 |
|------|-----|
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 流控 | 无 |

**帧结构**（总长度最大 264 字节）：

```
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
|  HEAD  |  SRC   |  DST   |  LEN   |  CMD   |  DATA[0..N] | TAIL0  | TAIL1  | TAIL2  | TAIL3  |
| 1 byte | 1 byte | 1 byte | 1 byte | 1 byte |   N bytes   | 1 byte | 1 byte | 1 byte | 1 byte |
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
```

- **HEAD**：固定 `0xAA`
- **LEN**：`CMD + DATA` 的总字节数，`N = LEN - 1`，范围 `1 ~ 255`
- **DATA**：数据域，长度由 LEN 推导
- **TAIL**：固定序列 `A5 5A FC FD`

**常用模块 ID**：

| ID | 模块 |
|----|------|
| `0x00` | Core（主控） |
| `0x40` | Submodel-1（UART6） |
| `0x41` | Submodel-2（UART7） |
| `0x42` | Submodel-3（UART8） |

> 完整协议规范（含接收状态机、通用操作码 `0x00~0x0F`、NACK 错误码定义）见 `PROJECT.md`。

## 2. 通用操作码定义

所有 Submodel 子类型共用以下操作码，通过 **DATA[0] 子命令** 区分具体功能。

| 操作码 | 宏名 | 方向 | 说明 |
| --- | --- | --- | --- |
| `0x40` | `CMD_SUB_EVT_NOTIFY` | Submodel → Core | 事件通知（识别成功、检测到 NFC 标签等） |
| `0x41` | `CMD_SUB_SET_MODE` | Core → Submodel | 设置模式 / 启动动作（注册指纹、开始测距等） |
| `0x42` | `CMD_SUB_GET_STATUS` | Core → Submodel | 查询状态 / 数据 |
| `0x43` | `CMD_SUB_DATA_REPORT` | Submodel → Core | 数据周期性上报（健康数据、RGB 状态等） |
| `0x44` | `CMD_SUB_SET_CONFIG` | Core → Submodel | 设置配置参数（删除指纹、删除标签等） |
| `0x45` | `CMD_SUB_ACTION_RESULT` | Submodel → Core | 动作结果（指纹注册成功、测距完成等） |
| `0x46` | `CMD_SUB_BULK_TRANSFER` | 双向 | 批量传输控制（副屏图片等） |
| `0x47~0x4F` | — | — | 预留 |

## 3. ACK/NACK 响应规则

### 3.1 设计原则

| 语义类别 | 说明 | 响应要求 |
|----------|------|---------|
| **查询 (Query)** | `GET_STATUS` | **必须 ACK + 数据** |
| **设置/动作 (Set/Action)** | `SET_MODE` / `SET_CONFIG` | **发送即忘，不响应** |
| **事件/上报 (Event)** | `EVT_NOTIFY` / `DATA_REPORT` / `ACTION_RESULT` | **不需要响应** |

> 发送即忘：Core 向 Submodel 发送设置/控制命令后不需要等待 ACK。

### 3.2 ACK / NACK 帧格式

**ACK 帧**：`[AA][DST][SRC][LEN][CMD=0x04][DATA...][A5][5A][FC][FD]`

**NACK 帧**：`[AA][DST][SRC][02][CMD=0x05][错误码:1][A5][5A][FC][FD]`

错误码详见 `PROJECT.md`。

## 4. 子类型协议详解

### 4.1 指纹识别（子类型 = 0x01）

#### 子命令定义

| 操作码 | 子命令 | 方向 | 说明 | DATA 格式 |
|--------|--------|------|------|----------|
| `0x41` | `0x01` | Core→FP | 开始注册新指纹 | `[子命令:1]` |
| `0x41` | `0x02` | Core→FP | 取消注册 | `[子命令:1]` |
| `0x44` | `0x01` | Core→FP | 删除指纹 | `[子命令:1][指纹ID:1]` |
| `0x44` | `0x02` | Core→FP | 清空所有指纹 | `[子命令:1]` |
| `0x42` | `0x00` | Core→FP | 查询指纹列表 | `[子命令:1]`，ACK + `[数量:1][[ID:1][名字:16字节]...]` |
| `0x40` | `0x01` | FP→Core | 识别成功 | `[子命令:1][指纹ID:1][名字:16字节]` |
| `0x40` | `0x02` | FP→Core | 识别失败 | `[子命令:1][错误码:1]` |
| `0x45` | `0x01` | FP→Core | 注册成功 | `[子命令:1][指纹ID:1][名字:16字节]` |
| `0x45` | `0x02` | FP→Core | 注册失败 | `[子命令:1][错误码:1]` |

> 名字字段固定 **16 字节**，不足补零，UTF-8 编码。

#### 注册流程

```
① Core → FP: [标准帧] CMD=0x41 SUB=0x01 开始注册
② FP → Core: [标准帧] CMD=0x04 ACK（进入注册模式）
③ 用户在指纹模块上录入指纹...
④ FP → Core: [标准帧] CMD=0x45 SUB=0x01 注册成功 + 新指纹ID + 名字
```

### 4.2 健康监测（子类型 = 0x02）

#### 子命令定义

| 操作码 | 子命令 | 方向 | 说明 | DATA 格式 |
|--------|--------|------|------|----------|
| `0x44` | `0x01` | Core→Health | 设置监测间隔 | `[子命令:1][间隔秒数:2(uint16大端)]` |
| `0x44` | `0x02` | Core→Health | 开始监测 | `[子命令:1]` |
| `0x44` | `0x03` | Core→Health | 停止监测 | `[子命令:1]` |
| `0x42` | `0x00` | Core→Health | 查询当前数据 | `[子命令:1]`，ACK + `[心跳:1][血氧:1][体温:1]` |
| `0x43` | `0x01` | Health→Core | 健康数据上报 | `[子命令:1][心跳:1][血氧:1][体温:1]` |

> **体温**：int8，°C，范围 -40~85。心率：0~255 BPM。血氧：0~100%。

### 4.3 NFC（子类型 = 0x03）

#### 子命令定义

| 操作码 | 子命令 | 方向 | 说明 | DATA 格式 |
|--------|--------|------|------|----------|
| `0x44` | `0x01` | Core→NFC | 删除标签 | `[子命令:1][标签ID:1]` |
| `0x44` | `0x02` | Core→NFC | 清空所有标签 | `[子命令:1]` |
| `0x42` | `0x00` | Core→NFC | 查询标签列表 | `[子命令:1]`，ACK + `[数量:1][[ID:1][UID:7字节]...]` |
| `0x40` | `0x01` | NFC→Core | 检测到标签 | `[子命令:1][标签ID:1][UID:7字节]` |

> UID 固定 7 字节（MIFARE Classic 兼容）。

### 4.4 触摸圆环 / 旋钮（子类型 = 0x04）

#### 子命令定义

| 操作码 | 子命令 | 方向 | 说明 | DATA 格式 |
|--------|--------|------|------|----------|
| `0x43` | `0x01` | Touch→Core | 旋钮状态 | `[子命令:1][旋钮值:2(uint16大端)]` |
| `0x43` | `0x02` | Touch→Core | 触摸事件 | `[子命令:1][X:2(uint16大端)][Y:2(uint16大端)][状态:1]` |
| `0x43` | `0x03` | Touch→Core | 模拟输入事件 | `[子命令:1][输入类型:1][键码:1][值:2(uint16大端)]` |

> **旋钮值**：0~65535，循环计数（增量编码器）。
> **触摸状态**：`0x00`=按下，`0x01`=移动，`0x02`=释放。
> **坐标系**：左上角为原点 (0,0)，X 向右，Y 向下，16 位归一化坐标。

#### 模拟输入事件详解

触摸圆环可作为类似 Keyboard 的输入设备，通过 `0x43 SUB=0x03` 上报模拟输入事件：

| 输入类型 | 说明 | 键码范围 | 值含义 |
|----------|------|---------|--------|
| `0x00` | 按键 | HID Usage ID | `0x0000`=释放, `0x0001`=按下 |
| `0x01` | 相对轴 | 轴编号 (0=X, 1=Y, 2=滚轮) | 有符号相对偏移量 |
| `0x02` | 绝对轴 | 轴编号 | 绝对坐标值 (0~65535) |

> 模拟输入事件由 Core 转换为标准 HID 报告后通过 CH9350 或蓝牙发送，与 Keyboard 模块的按键处理流程一致。

### 4.5 RGB 灯效（子类型 = 0x05）

RGB 模块为 **7×7 WS2812 LED 矩阵**（共 49 颗灯珠），支持预设模式与自定义逐帧动画。

#### 子命令定义

| 操作码 | 子命令 | 方向 | 说明 | DATA 格式 |
|--------|--------|------|------|----------|
| `0x41` | `0x01` | Core→RGB | 设置 RGB 模式 | `[子命令:1][模式:1][R:1][G:1][B:1][亮度:1][速度:1]` |
| `0x41` | `0x02` | Core→RGB | 传输自定义帧 | `[子命令:1][帧序号:1][帧数据:147]` |
| `0x41` | `0x03` | Core→RGB | 播放自定义动画 | `[子命令:1][总帧数:1][帧间隔:2(uint16大端)]` |
| `0x42` | `0x00` | Core→RGB | 查询当前模式 | `[子命令:1]`，ACK + `[模式:1][R:1][G:1][B:1][亮度:1][速度:1]` |
| `0x43` | `0x01` | RGB→Core | 当前模式上报 | `[子命令:1][模式:1][R:1][G:1][B:1][亮度:1][速度:1]` |

> **模式**：`0x00`=自定义（从 rgb.json 加载），`0x01`=常亮，`0x02`=呼吸，`0x03`=跑马灯。
> **帧间隔**：每帧之间的延迟时间，单位 ms。

#### 预设模式说明

| 模式 | 名称 | 行为 |
|------|------|------|
| `0x00` | 自定义 | 从 `rgb.json` 加载逐帧动画数据 |
| `0x01` | 常亮 | 全部 LED 以指定 R/G/B 颜色常亮 |
| `0x02` | 呼吸 | 全部 LED 以指定颜色呼吸，速度参数控制周期 |
| `0x03` | 跑马灯 | 单灯依次点亮，速度参数控制移动速率 |

#### 自定义帧动画协议

自定义模式（`rgb_mode=0`）下，Core 从 CONFIG 目录下的 `rgb.json` 读取逐帧动画数据，通过标准帧逐帧传输给 RGB 模块。

**rgb.json 格式**：

```json
{
  "frame_count": 5,
  "frame_interval": 100,
  "frames": [
    [[0,0,0],[0,0,0],[0,0,0],...],
    [[255,0,0],[0,255,0],[0,0,255],...],
    ...
  ]
}
```

- `frame_count`：总帧数，最大 **20** 帧
- `frame_interval`：帧间隔（ms），范围 50~1000
- `frames`：数组，每帧包含 49 个 RGB 三元组（每颗 LED 的 24bit RGB888 颜色值）
- 每颗 LED 用 `[R, G, B]` 表示，如 `[255, 0, 0]` 为纯红
- 每帧数据按行优先排列：`[LED0, LED1, ..., LED48]`，对应矩阵位置 `(row, col) = index / 7, index % 7`

**帧传输流程**：

```
① Core 读取 rgb.json，解析帧数据
② Core → RGB: [标准帧] CMD=0x41 SUB=0x02 传输第 N 帧
   DATA: [帧序号:1][LED0_R:1][LED0_G:1][LED0_B:1]...[LED48_R:1][LED48_G:1][LED48_B:1]
   帧序号从 0 开始，每帧 DATA 共 149 字节（1+147，其中 147=49×3 RGB888）

③ 重复②直到所有帧传输完毕

④ Core → RGB: [标准帧] CMD=0x41 SUB=0x03 开始播放
   DATA: [总帧数:1][帧间隔:2(uint16大端)]

⑤ RGB 模块循环播放已接收的帧动画
```

> **帧序号**：0~19，最多 20 帧。
> **单帧数据量**：147 字节（49×3 RGB888），加帧序号共 148 字节，在标准帧最大 255 字节数据域内。
> **RGB888 颜色**：每颗 LED 用 3 字节表示完整颜色（R, G, B 各 1 字节），颜色由帧数据决定，亮度由模式设置命令的全局亮度参数控制。
> **HSV 饱和度**：渲染时 HSV 饱和度强制为最大值（255），仅通过 V 分量缩放实现亮度控制。

### 4.6 红外测距（子类型 = 0x06）

#### 子命令定义

| 操作码 | 子命令 | 方向 | 说明 | DATA 格式 |
|--------|--------|------|------|----------|
| `0x41` | `0x01` | Core→IR | 开始测距 | `[子命令:1]` |
| `0x41` | `0x02` | Core→IR | 停止测距 | `[子命令:1]` |
| `0x45` | `0x01` | IR→Core | 测距结果 | `[子命令:1][距离:2(uint16大端)][单位:1][精度:1]` |
| `0x45` | `0x02` | IR→Core | 测距失败 | `[子命令:1][错误码:1]` |

> **单位**：`0x00`=mm，`0x01`=cm，`0x02`=m。
> **精度**：测量精度百分比（如 `0x05`=±5%）。

#### 测距流程

```
① Core → IR: [标准帧] CMD=0x41 SUB=0x01 开始测距
② IR → Core: [标准帧] CMD=0x04 ACK（进入测距模式）
③ 红外模块完成测距...
④ IR → Core: [标准帧] CMD=0x45 SUB=0x01 测距结果 + 距离 + 单位 + 精度
```

### 4.7 副屏（子类型 = 0x07）

副屏为反射式黑白屏，分辨率较低（如 200×200 或 400×300），通过 UART 接收 Core 发送的状态信息和背景图片。

#### 子命令定义

| 操作码 | 子命令 | 方向 | 说明 | DATA 格式 |
|--------|--------|------|------|----------|
| `0x41` | `0x01` | Core→SubDisp | 传输模块状态 | `[子命令:1][状态数据:变长]` |
| `0x41` | `0x02` | Core→SubDisp | 传输自定义内容 | `[子命令:1][内容类型:1][内容数据:变长]` |
| `0x41` | `0x03` | Core→SubDisp | 清屏 | `[子命令:1]` |
| `0x46` | `0x01` | Core→SubDisp | 图片传输握手 | `[子命令:1][图片宽度:2][图片高度:2][数据大小:4]` |
| `0x46` | `0x02` | SubDisp→Core | 图片接收完成确认 | `[子命令:1][结果:1]` |

#### 状态数据格式（CMD 0x41 SUB=0x01）

```
DATA[1]: 状态位图（1字节）
  - bit0: 电量有效    - bit1: 蓝牙状态有效
  - bit2: 时间有效    - bit3: 当前应用有效
DATA[2]: 电量百分比（1字节，若 bit0=1）
DATA[3]: 蓝牙状态（1字节，0=断开, 1=已连接，若 bit1=1）
DATA[4..7]: 时间戳（Unix 时间秒数，32位大端，若 bit2=1）
DATA[8]: 当前应用ID（1字节，若 bit3=1）
```

> 状态数据长度由 LEN 推导，位图指示哪些字段有效。

#### 自定义内容格式（CMD 0x41 SUB=0x02）

```
DATA[1]: 内容类型（1字节）
  - 0x00: 文本    - 0x01: 图标    - 0x02: 进度条
  - 0x03: 简单图形（矩形/圆）
DATA[2..N]: 内容数据（变长，由内容类型定义）
```

#### 图片传输流程（Bulk Mode）

副屏图片为黑白位图（1 bit/像素），采用 **标准帧握手 + Raw 数据传输**：

```
① Core → SubDisp: [标准帧] CMD=0x46 SUB=0x01 图片握手
   DATA: [宽度:2][高度:2][总字节数:4]
   总字节数 = 宽度 × 高度 / 8（向上取整）

② SubDisp → Core: [标准帧] CMD=0x04 ACK（准备接收）

③ [进入 Bulk Mode - 数据阶段]
   Core → SubDisp: [Raw] 黑白位图原始数据（无协议头尾）

④ SubDisp → Core: [标准帧] CMD=0x46 SUB=0x02 接收完成确认
   DATA[0]: 结果（0=成功, 1=失败）
```

> 图片数据按行优先排列，每行字节数 = `(宽度 + 7) / 8`。bit0 对应每字节最左边的像素。

## 5. CMD_GET_TYPE 响应

Submodel 响应 `CMD_GET_TYPE` 时，`CMD_ACK` 的 DATA 格式如下：

| 字节 | 字段 | 值/说明 |
|------|------|---------|
| DATA[0] | 模块类型编号 | `0x05` |
| DATA[1] | 模块子类型编号 | 见下表 |
| DATA[2] | 硬件版本号 | 模块硬件版本 |
| DATA[3] | 固件主版本号 | 固件主版本 |
| DATA[4] | 固件次版本号 | 固件次版本 |
| DATA[5..N] | 扩展信息 | 能力位（可选） |

**Submodel 子类型定义**

> 代码宏定义：`protocol.h` → `MODULE_SUBTYPE_SUBMODEL_*`

| 子类型编号 | 宏名 | 实现名称 | 说明 |
|------------|------|----------|------|
| `0x00` | `MODULE_SUBTYPE_SUBMODEL_RESERVED` | 保留 | 系统保留 |
| `0x01` | `MODULE_SUBTYPE_SUBMODEL_FINGERPRINT` | 指纹识别 | 指纹录入与识别 |
| `0x02` | `MODULE_SUBTYPE_SUBMODEL_HEALTH` | 健康监测 | 心跳/血氧/体温 |
| `0x03` | `MODULE_SUBTYPE_SUBMODEL_NFC` | NFC 读卡 | 标签识别与管理 |
| `0x04` | `MODULE_SUBTYPE_SUBMODEL_TOUCH_RING` | 触摸圆环 | 旋钮 + 触摸输入 |
| `0x05` | `MODULE_SUBTYPE_SUBMODEL_RGB` | RGB 灯效 | 灯效控制 |
| `0x06` | `MODULE_SUBTYPE_SUBMODEL_INFRARED` | 红外测距 | 距离测量 |
| `0x07` | `MODULE_SUBTYPE_SUBMODEL_SUB_DISPLAY` | 副屏 | 反射式黑白屏 |
| `0x08~0xFF` | — | 预留 | 未来扩展 |
