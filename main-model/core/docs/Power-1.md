# Power-1 模块（供电管理）

## 概述

本模块为核心底板及所有外设提供稳定 5V 供电，同时承担电池管理、充放电控制及功耗监测功能。

- **主控芯片**：CH32V103C8T6
- **电池**：6000mAh 锂电池
- **与核心模块接口**：模块侧 UART1，**115200/8-N-1**
- **模块 ID**：`0x30`
- **模块类型编号**：`0x03`
- **子类型编号**：`0x01`（标准供电模块）

## 硬件组成

- **CH32V103C8T6**：电池管理逻辑、充放电控制、UART 通信
- **电池管理芯片 / 电量计**：监测电池电压、电流、温度、剩余容量（具体型号根据硬件设计确定，如 CW2015 / BQ40z50 等）
- **充电芯片**：支持 PD 快充协议与无线充电接收协议（如 IP2312 / BQ25890 等）
- **无线充电线圈与接收电路**（若支持无线充电）
- **USB-C 接口**：电源输入（PD 快充）与 5V 慢充输出（当充电宝使用）
- **DC-DC / LDO**：将电池电压稳压至 5V，输出给核心板及其他模块

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、ADC、UART1（115200）
- 初始化电池管理芯片：配置电量计参数、充电电流限制、温度保护阈值
- 初始化 PD 快充芯片：协商电压档位（通常为 5V/9V/12V，最终降压到电池充电电压）
- 读取当前电池状态（电压、电量百分比、温度），作为初始上报值

### 运行阶段

- **电量监测**：定时采样电池电压、电流、温度，计算剩余容量百分比（SOC）
- **充电管理**：检测 USB-C 插入 / 拔出、PD 协议握手、充电电流动态调节；充满后自动切换涓流或停止充电
- **放电管理**：监测输出电流与电池温度，过流 / 过热时切断输出保护；支持作为充电宝对外慢充输出
- **状态上报**：定时或通过变化触发，向 Core 上报：
  - 电池电量百分比（`CMD_PWR_BAT_LOW_EVT` 低电量告警）
  - 充电状态变化（插入 / 拔出 / 充满，`CMD_PWR_CHARGE_EVT`）
  - 当前电压、电流、功率（响应 `CMD_PWR_GET_VOLTAGE` 等查询）
- **休眠与唤醒**：接收 Core 的功耗策略指令，在系统 idle 时降低自身功耗；检测到充电插入或按键唤醒信号时恢复全速运行

### 中断与事件

- ADC / 电量计中断：电池参数采样完成
- GPIO 中断：USB-C 插入 / 拔出检测、无线充电状态变化
- 定时器中断：周期性电量监测与状态上报
- UART1 中断：接收 Core 的查询或配置指令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`battery.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Battery_Get_Soc()`
- **变量**：统一小写加下划线，例如 `bat_voltage`、`charge_status`
- **文件夹**：下划线写法，仅首字母大写，例如 `Battery`、`Charger`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **安全第一**：电池管理涉及过充、过放、过流、过热等风险，硬件层面必须有多重保护（保护板 / PCM），软件层面做二次监测与告警。
2. **电量精度**：若使用 ADC 分压测电压估算 SOC，需根据电池放电曲线做多点校准；若使用专用电量计芯片，需按手册配置电池化学 ID 与设计容量。
3. **PD 快充协议**：PD 握手由专用芯片完成时，CH32V103 只需通过 I2C 读取芯片状态；若需软件实现 PD 物理层，工作量极大，不建议。
4. **对外慢充输出**：作为充电宝对外输出 5V 时，需限流保护（建议 1A 以内），防止电池过放或过热。
5. **低功耗状态**：Power 模块是整机电源源头，即使系统休眠也需保持最小监测功能；CH32V103 可在浅睡模式下定时唤醒采样，降低待机电流。
6. **与 Core 的协同**：Core 侧根据电量决定性能策略（如低电量时降频、关闭背光、暂停非必要模块），Power 模块只负责如实上报数据，策略决策在 Core。
