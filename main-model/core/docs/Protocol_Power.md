# Power 模块通信协议

## 1. 模块概述

- **模块类型编号**：`0x03`
- **模块 ID**：`0x30`（预留范围 `0x30~0x31`）
- **物理接口**：UART5
- **主要交互核心**：V3F
- **说明**：V3F 直接管理供电模块的 UART 中断与协议解析，负责电池状态监控和充电策略控制。

## 2. 操作码定义

| 操作码 | 宏名 | 方向 | 说明 |
| --- | --- | --- | --- |
| `0x31` | `CMD_PWR_GET_BATTERY` | Core -> Power | 查询电池电量百分比。 |
| `0x32` | `CMD_PWR_GET_VOLTAGE` | Core -> Power | 查询电池电压（mV）。 |
| `0x33` | `CMD_PWR_SET_CHARGE` | Core -> Power | 设置充电策略。DATA[0] = 开关/电流档位。 |
| `0x34` | `CMD_PWR_BAT_LOW_EVT` | Power -> Core | 低电量告警事件。DATA[0] = 当前电量。 |
| `0x35` | `CMD_PWR_CHARGE_EVT` | Power -> Core | 充电状态变化事件（插入/拔出/充满）。 |

## 3. CMD_GET_TYPE 响应

Power 模块响应 `CMD_GET_TYPE` 时，`CMD_ACK` 的 DATA 格式如下：

| 字节 | 字段 | 值/说明 |
|------|------|---------|
| DATA[0] | 模块类型编号 | `0x03` |
| DATA[1] | 模块子类型编号 | `0x01`（标准供电模块） |
| DATA[2] | 硬件版本号 | 电源管理硬件版本 |
| DATA[3] | 固件主版本号 | 电源管理固件主版本 |
| DATA[4] | 固件次版本号 | 电源管理固件次版本 |
| DATA[5..N] | 扩展信息 | 电池类型、最大充电电流等能力位（可选） |

> 当前 Power 仅规划一种实现，子类型固定为 `0x01`，预留 `0x02~0xFF` 用于未来扩展。 |
