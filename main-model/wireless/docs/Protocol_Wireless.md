# Wireless 模块通信协议

## 1. 模块概述

- **模块类型编号**：`0x02`
- **模块 ID**：`0x01`
- **物理接口**：SPI4（Core 侧 V5F 为 Master，CH585F 为 Slave）

CH585F 为独立 RISC-V MCU，内置 **BLE 协议栈**，与 Core 通过 SPI4 进行全双工通信。应用层载荷严格遵循统一紧凑二进制协议格式。

> **注意**：CH585F 仅支持 **BLE（低功耗蓝牙）**，不支持 Bluetooth Classic，因此不支持 A2DP、SPP 等 Classic 协议。蓝牙键鼠走 **HID over GATT**，APP 数据通道走 BLE 自定义 Profile / L2CAP。

### 1.1 帧格式速查

SPI4 物理层参数：

| 参数 | 值 |
|------|-----|
| 接口 | SPI4 |
| 角色 | V5F = Master，CH585F = Slave |
| 数据位 | 8 |
| 时钟极性 | CPOL Low |
| 时钟相位 | CPHA 1Edge |
| NSS | 软件控制 |
| 波特率 | 由 `CH585F_SPI_CLOCK` 分频决定 |

**帧结构**（总长度最大 264 字节）：

```
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
|  HEAD  |  SRC   |  DST   |  LEN   |  CMD   |  DATA[0..N] | TAIL0  | TAIL1  | TAIL2  | TAIL3  |
| 1 byte | 1 byte | 1 byte | 1 byte | 1 byte |   N bytes   | 1 byte | 1 byte | 1 byte | 1 byte |
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
```

- **HEAD**：固定 `0xAA`。仅在接收状态机处于“等待帧头”时识别为新帧起始，数据域中的 `0xAA` 不会触发重解析。
- **LEN**：`CMD + DATA` 的总字节数，即 `N = LEN - 1`，范围 `1 ~ 255`。`LEN = 1` 表示 DATA 为空。
- **DATA**：数据域，长度由 LEN 推导，最大 255 字节。
- **TAIL**：固定序列 `A5 5A FC FD`，任一字节不匹配则丢弃整帧。

**常用模块 ID**：

| ID | 模块 |
|----|------|
| `0x00` | Core（主控） |
| `0x01` | Wireless / CH585F（本模块） |

**帧长度计算**：
```
总帧长 = 1(HEAD) + 1(SRC) + 1(DST) + 1(LEN) + 1(CMD) + N(DATA) + 4(TAIL)
       = 9 + N
       = 8 + LEN
```

> 完整协议规范（含接收状态机、通用操作码 `0x00~0x0F`、NACK 错误码定义）见 `PROJECT.md`。

### 1.2 SPI 通信特殊说明

CH585F 作为 **SPI Slave**，无法主动发起数据传输。所有上行数据（CH585F → Core）均需依赖以下机制之一：

1. **中断线通知（推荐）**：CH585F 通过独立 GPIO 中断线（如 `INT`）通知 V5F"有数据待读取"，V5F 收到中断后发起 SPI 读取事务。
2. **轮询（备用）**：V5F 主循环中定期发送查询帧（如 `CMD_BT_GET_STATUS` 或空操作），CH585F 在响应中夹带上行数据。
3. **SPI 全双工复用**：V5F 每次发送下行帧时，同步接收 MISO 上的数据；CH585F 利用此窗口将上行数据填入发送缓冲区。

> **建议**：硬件上预留中断线（如 PE10 或其他空闲 GPIO），否则高频 HID 事件场景下轮询开销较大。

## 2. 操作码定义

Wireless 模块专用操作码范围为 `0x50 ~ 0x5F`（特例，不参与模块 ID 高 4 位映射）。当基础操作码空间不足时，使用扩展操作码机制。

### 2.1 基础操作码（0x51 ~ 0x5F）

| 操作码 | 宏名 | 方向 | 说明 | DATA 格式 |
| --- | --- | --- | --- | --- |
| `0x51` | `CMD_BT_GET_STATUS` | Core -> CH585F | 查询蓝牙连接状态 | 无，CH585F 回复 ACK + 状态数据 |
| `0x52` | `CMD_BT_SEND_DATA` | Core -> CH585F | 通过蓝牙发送原始数据（APP 数据通道） | `[数据:变长]` |
| `0x53` | — | — | 预留 | — |
| `0x54` | `CMD_BT_CONN_EVT` | CH585F -> Core | 蓝牙连接/断开/配对事件 | `[事件类型:1][设备类型:1][MAC:6][名称:16]` |
| `0x55` | `CMD_BT_RECV_DATA` | CH585F -> Core | 蓝牙接收到数据上报（APP 数据通道） | `[数据:变长]` |
| `0x56` | `CMD_BT_HID_REPORT` | CH585F -> Core | HID 输入事件上报（BLE 键鼠） | `[报告类型:1][HID数据:变长]` |
| `0x57` | — | — | 预留 | — |
| `0x58` | `CMD_BT_SET_DISCOVERABLE` | Core -> CH585F | 设置可发现性/广播开关 | `[开关:1]` |
| `0x59` | `CMD_BT_RESET` | Core -> CH585F | 复位蓝牙模块 | `[确认码:0xA5]` |
| `0x5A~0x5F` | — | — | 预留 | — |

### 2.2 扩展操作码（CMD = 0x50，DATA[0] 为子命令）

| 扩展码 | 宏名 | 方向 | 说明 | DATA 格式 |
| --- | --- | --- | --- | --- |
| `0x01` | `CMD_BT_EXT_SCAN` | Core -> CH585F | 请求扫描/停止扫描 | `[操作:1][超时:1][过滤:1]` |
| `0x02` | `CMD_BT_EXT_DEVICE_LIST` | CH585F -> Core | 设备列表（扫描结果/配对列表） | `[列表类型:1][数量:1][设备信息:23字节×N]` |
| `0x03` | `CMD_BT_EXT_CONNECT` | Core -> CH585F | 请求连接/断开指定设备 | `[操作:1][MAC:6]` |
| `0x04` | `CMD_BT_EXT_PAIRING_MGMT` | 双向 | 配对设备管理（查询/删除/清空/结果） | `[操作:1][MAC(6)/结果(1)]` |
| `0x05` | `CMD_BT_EXT_SET_MODE` | Core -> CH585F | 设置蓝牙工作模式与广播参数 | `[模式:1][名称开关:1][自定义名:16]` |
| `0x06` | — | — | 预留 | — |
| `0x07` | `CMD_BT_EXT_CLI_DATA` | 双向 | APP CLI 数据通道 | `[数据:变长]` |
| `0x08` | — | — | 预留 | — |
| `0x09` | — | — | 预留 | — |
| `0x0A~0x3F` | — | — | 预留 | — |

## 3. ACK/NACK 响应规则

### 3.1 设计原则

ACK/NACK 由**命令语义**决定：

| 语义类别 | 说明 | 响应要求 |
|----------|------|---------|
| **查询 (Query)** | 请求获取某信息 | **必须 ACK + 数据** |
| **动作 (Action)** | 请求执行某操作 | **必须 ACK（空）或 NACK** |
| **设置 (Set)** | 修改某参数/状态 | **发送即忘，不响应** |
| **事件 (Event)** | 单向状态/输入上报 | **不需要响应** |

> **发送即忘**：Core 向 CH585F 发送设置类命令后不需要等待 ACK。CH585F 被动执行，不支持时静默丢弃。

### 3.2 操作码响应要求

| 码 | 命令 | 语义 | 接收方行为 |
|---|---|---|---|
| `0x51` | GET_STATUS | 查询 | CH585F 回复 ACK + 状态数据 |
| `0x52` | SEND_DATA | 动作 | CH585F 回复 ACK 空 或 NACK（缓冲区满时） |
| `0x53` | — | — | 预留 |
| `0x54` | CONN_EVT | 事件 | Core 不回复 |
| `0x55` | RECV_DATA | 事件 | Core 不回复 |
| `0x56` | HID_REPORT | 事件 | Core 不回复 |
| `0x57` | — | — | 预留 |
| `0x58` | SET_DISCOVERABLE | 设置 | 发送即忘，CH585F 不回复 |
| `0x59` | RESET | 动作 | CH585F 回复 ACK 空 或 NACK |
| `0x50/0x01` | SCAN | 动作 | CH585F 回复 ACK 空 或 NACK |
| `0x50/0x02` | DEVICE_LIST | 查询响应 | 发送即忘（作为扫描/查询的 ACK payload） |
| `0x50/0x03` | CONNECT | 动作 | CH585F 回复 ACK 空 或 NACK |
| `0x50/0x04` | PAIRING_MGMT | 动作/查询 | 双向 ACK 或 NACK |
| `0x50/0x05` | SET_MODE | 设置 | 发送即忘，CH585F 不回复 |
| `0x50/0x06` | — | — | 预留 |
| `0x50/0x07` | CLI_DATA | 数据通道 | **应用层自定义**，协议层不强制 ACK |
| `0x50/0x08` | — | — | 预留 |
| `0x50/0x09` | — | — | 预留 |

### 3.3 ACK / NACK 帧格式

**ACK 帧**：
```
[AA][DST][SRC][LEN][CMD=0x04][DATA...][A5][5A][FC][FD]
```
- SRC/DST 与原请求**交换**
- DATA 可选携带响应数据，`LEN = 1 + data_len`
- 空 ACK：`LEN = 01`，无 DATA

**NACK 帧**：
```
[AA][DST][SRC][02][CMD=0x05][错误码:1][A5][5A][FC][FD]
```
- DATA[0] = 错误码（1 字节），详见 `PROJECT.md` 通用协议错误码定义
- `LEN = 02`

### 3.4 Core 侧超时处理

Core 发送**查询/动作类**命令后，应在 **500ms** 内收到 ACK/NACK。超时后由应用层处理。**协议层不实现自动重传**。

## 4. 数据格式详解

### 4.1 蓝牙状态格式（CMD 0x51 ACK）

CH585F 回复 `CMD_BT_GET_STATUS` 时使用以下格式：

```
DATA[0]: 状态位图（1字节）
  - bit0: 蓝牙开关状态（0=关闭, 1=开启）
  - bit1: 是否正在扫描（0=否, 1=是）
  - bit2: 是否有 BLE 设备已连接（0=否, 1=是）
  - bit3: APP 数据通道是否已连接（0=否, 1=是）
  - bit4~7: 预留

DATA[1]: 已连接设备数量（1字节）
DATA[2]: 当前工作模式（1字节）
  - 0x00: 待机/关闭
  - 0x01: HID Host 模式（扫描连接 BLE 键鼠）
  - 0x02: Peripheral 模式（广播等待 APP 作为 Central 连接）
  - 0x03: 并发模式（HID + APP 同时运行）

DATA[3..8]: 主连接设备 MAC 地址（6字节，无连接时全填 0x00）
```

> 总 DATA 长度：9 字节，`LEN = 10`。

### 4.2 蓝牙数据发送格式（CMD 0x52）

Core 通过本命令向已连接的 BLE 设备（如手机 APP）发送原始数据。

```
DATA[0..N]: 原始数据（变长，N ≤ 255，实际受 LEN 限制）
```

> 本命令为通用数据通道，不解析数据内容。若目标为 APP CLI，建议优先使用扩展码 `0x07`（`CMD_BT_EXT_CLI_DATA`）以获得更清晰的语义。

### 4.3 连接事件格式（CMD 0x54）

CH585F 在 BLE 设备连接、断开、配对状态变化时主动上报。

```
DATA[0]: 事件类型（1字节）
  - 0x00: 设备已连接
  - 0x01: 设备已断开
  - 0x02: 连接失败/超时
  - 0x03: 配对成功
  - 0x04: 配对请求（需用户确认）

DATA[1]: 设备类型（1字节）
  - 0x00: 未知
  - 0x01: BLE 键盘
  - 0x02: BLE 鼠标
  - 0x03: 其他 BLE HID 设备
  - 0x04: 手机/PC（APP）

DATA[2..7]: 设备 MAC 地址（6字节）
DATA[8..23]: 设备名称（16字节，不足补 0x00，UTF-8 编码）
```

> 总 DATA 长度：24 字节，`LEN = 25`。

### 4.4 蓝牙数据接收格式（CMD 0x55）

CH585F 收到 BLE 远端（如手机 APP）发来的数据后，通过本命令上报 Core。

```
DATA[0..N]: 原始数据（变长）
```

### 4.5 HID 报告格式（CMD 0x56）

CH585F 在连接的 BLE HID 设备（键盘/鼠标）有输入时上报。格式与 Keyboard 模块的 `CMD_KBD_HID_REPORT` 及 Display 模块的统一输入事件兼容，便于 Core 直接转发。

> **注意**：CH585F 仅支持 **BLE HID over GATT (HOGP)**，不支持 Bluetooth Classic HID。

```
DATA[0]: 报告类型（1字节）
  - 0x00: 键盘报告（仿 HID Keyboard Boot Report）
  - 0x01: 鼠标报告（仿 HID Mouse Boot Report）

DATA[1..N]: HID 报告数据（N = LEN - 2）
```

**键盘报告（类型 = 0x00，N = 8）**：
```
DATA[1]: 修饰键（bitmask）
  - bit0: Left Ctrl    - bit1: Left Shift
  - bit2: Left Alt     - bit3: Left GUI
  - bit4: Right Ctrl   - bit5: Right Shift
  - bit6: Right Alt    - bit7: Right GUI
DATA[2]: 保留（0x00）
DATA[3]: 键码 0（USB HID Usage ID，0x00 表示空槽）
DATA[4]: 键码 1
DATA[5]: 键码 2
DATA[6]: 键码 3
DATA[7]: 键码 4
DATA[8]: 键码 5
```

> 最多同时上报 6 个非修饰键。释放按键时对应键码槽填 `0x00`。

**鼠标报告（类型 = 0x01，N = 4）**：
```
DATA[1]: 按钮状态（bitmask）
  - bit0: 左键    - bit1: 右键    - bit2: 中键
DATA[2]: X 轴偏移（int8，-128~127，正值向右）
DATA[3]: Y 轴偏移（int8，-128~127，正值向下）
DATA[4]: 滚轮偏移（int8，正值为远离用户/向上滚动）
```

### 4.6 扫描请求格式（扩展码 0x01）

Core 请求 CH585F 开始或停止扫描周围 BLE 设备。

```
DATA[0]: 基础操作码 = 0x50
DATA[1]: 扩展码 = 0x01
DATA[2]: 操作类型（1字节）
  - 0x00: 开始扫描
  - 0x01: 停止扫描

DATA[3]: 扫描超时（1字节，单位秒，0 表示使用默认 10 秒）
DATA[4]: 设备类型过滤（1字节，bitmask）
  - bit0: HID 设备（键鼠）
  - bit1: 手机/PC（APP）
  - bit2~7: 预留
  - 0xFF: 不过滤（扫描所有 BLE 设备）
```

> 总 DATA 长度：5 字节，`LEN = 6`。

### 4.7 设备列表格式（扩展码 0x02）

CH585F 将扫描结果或已配对设备列表上报 Core。本格式与 Display 模块的蓝牙设备列表（`Protocol_Display.md` 5.9 节）保持一致，方便 Core 直接转发给 Display。

```
DATA[0]: 基础操作码 = 0x50
DATA[1]: 扩展码 = 0x02
DATA[2]: 列表类型（1字节）
  - 0x00: 扫描结果
  - 0x01: 已配对设备列表

DATA[3]: 设备数量 N（1字节）
DATA[4]:      设备1状态（1字节）
  - 0x00: 已配对
  - 0x01: 已连接
  - 0x02: 发现中（仅扫描结果）
DATA[5..20]:  设备1名称（16字节，不足补 0x00，UTF-8 编码）
DATA[21..26]: 设备1 MAC 地址（6字节）
... 后续设备
```

> 每个设备信息固定 **23 字节**：状态(1) + 名称(16) + MAC(6)。
> 设备数量 `N = (LEN - 1 - 3) / 23`。
> 若设备列表超过单帧容量，可分多帧上报，或先以 `NACK`（`PROTO_ERR_LEN_MISMATCH`）拒绝，再由 Core 通过 `CMD_BT_EXT_PAIRING_MGMT`（查询）分批次获取。

### 4.8 连接/断开请求格式（扩展码 0x03）

Core 请求 CH585F 连接或断开指定 BLE 设备。

```
DATA[0]: 基础操作码 = 0x50
DATA[1]: 扩展码 = 0x03
DATA[2]: 操作类型（1字节）
  - 0x00: 断开指定设备
  - 0x01: 连接指定设备（需已配对或在最近扫描结果中）
  - 0x02: 断开所有已连接设备

DATA[3..8]: 目标设备 MAC 地址（6字节，操作类型为 0x00/0x01 时有效；0x02 时可填 0x00）
```

> 总 DATA 长度：9 字节，`LEN = 10`。

### 4.9 配对管理格式（扩展码 0x04）

双向命令，用于管理已配对设备列表。

**查询/删除/清空（Core -> CH585F）**：
```
DATA[0]: 基础操作码 = 0x50
DATA[1]: 扩展码 = 0x04
DATA[2]: 操作类型（1字节）
  - 0x00: 查询已配对列表
  - 0x01: 删除指定配对设备
  - 0x02: 清空所有配对记录

DATA[3..8]: MAC 地址（6字节，仅操作类型 0x01 时有效）
```

- 操作类型 `0x00`：CH585F 回复 ACK，其 DATA 使用扩展码 `0x02`（`CMD_BT_EXT_DEVICE_LIST`，列表类型 = `0x01`）。
- 操作类型 `0x01/0x02`：发送即忘，或回复 ACK 空。

**配对结果上报（CH585F -> Core）**：
```
DATA[0]: 基础操作码 = 0x50
DATA[1]: 扩展码 = 0x04
DATA[2]: 操作类型 = 0x03（配对结果）
DATA[3]: 结果状态（1字节）
  - 0x00: 配对成功
  - 0x01: 配对失败（拒绝/超时）
  - 0x02: 配对取消

DATA[4..9]: 设备 MAC 地址（6字节）
DATA[10..25]: 设备名称（16字节）
```

### 4.10 工作模式设置格式（扩展码 0x05）

Core 设置 CH585F 的蓝牙工作模式与广播参数。

```
DATA[0]: 基础操作码 = 0x50
DATA[1]: 扩展码 = 0x05
DATA[2]: 工作模式（1字节）
  - 0x00: 关闭蓝牙
  - 0x01: HID Host 模式（扫描连接 BLE 键鼠，不对外广播等待 APP）
  - 0x02: Peripheral 模式（广播等待 APP 作为 Central 连接）
  - 0x03: 双角色并发模式（Central 连接 BLE 键鼠 + Peripheral 被手机 APP 连接）

DATA[3]: 广播名称开关（1字节）
  - 0x00: 不修改广播名称
  - 0x01: 使用默认广播名称
  - 0x02: 使用后续自定义名称

DATA[4..19]: 自定义广播名称（16字节，不足补 0x00，仅当 DATA[3]=0x02 时有效）
```

> 总 DATA 长度：20 字节，`LEN = 21`。

### 4.11 CLI 数据格式（扩展码 0x07）

移动端 APP 通过 BLE 自定义 Profile（如 Nordic UART Service 或自定义 UUID）与 CH585F 建立数据通道，CH585F 通过本扩展码与 Core 双向透传 CLI 数据。

```
DATA[0]: 基础操作码 = 0x50
DATA[1]: 扩展码 = 0x07
DATA[2..N]: CLI 原始数据（变长，UTF-8 编码）
```

**数据流向**：
- **下行**（APP -> CH585F -> Core）：APP 发送命令字符串，CH585F 通过本命令上报，Core 转发给 `CLI_Process()` 解析执行。
- **上行**（Core -> CH585F -> APP）：`CLI_Process()` 的输出通过 `printf()` 经本命令下发给 CH585F，由 CH585F 经 BLE 回传 APP。

> 建议单帧 CLI 数据不超过 200 字节（为 LEN 和协议头尾留余量）。超长输出应分帧发送。

## 5. CMD_GET_TYPE 响应

CH585F 模块响应 `CMD_GET_TYPE` 时，`CMD_ACK` 的 DATA 格式如下：

| 字节 | 字段 | 值/说明 |
|------|------|---------|
| DATA[0] | 模块类型编号 | `0x02` |
| DATA[1] | 模块子类型编号 | `0x01`（`MODULE_SUBTYPE_WIRELESS_BT`，标准 BLE 模块） |
| DATA[2] | 硬件版本号 | CH585F 硬件版本 |
| DATA[3] | 固件主版本号 | BLE 协议栈或固件主版本 |
| DATA[4] | 固件次版本号 | BLE 协议栈或固件次版本 |
| DATA[5..N] | 扩展信息 | BLE 版本、支持协议（HID over GATT / 自定义 Profile）等能力位（可选） |

**Wireless 子类型定义**

> 代码宏定义：`protocol.h` → `MODULE_SUBTYPE_WIRELESS_*`

| 子类型编号 | 宏名 | 实现名称 | 说明 |
|------------|------|----------|------|
| `0x00` | `MODULE_SUBTYPE_WIRELESS_RESERVED` | 保留 | 系统保留 |
| `0x01` | `MODULE_SUBTYPE_WIRELESS_BT` | 标准 BLE 模块 | CH585F（BLE 5.4，支持 HID over GATT + 自定义数据通道） |
| `0x02~0xFF` | — | 预留 | 未来扩展 |
