# 项目结构
这是一个双核项目，主要在hardware.c中完成V5F作为主要执行核心，V3F负责唤醒V5F和串口转发。

将驱动初始化相关任务集合到hardware.h，主要包括CS43131、CH378,CH585F自主初始化。

这两项初始化任务均由V5F执行，后续任务也由其执行。

V3F需要初始化八个串口并执行扫描当前模块组成情况并完成hardware.c中的全局配件状态的更新，设置系统初始化标志位0。V5F主要负责板上系统的初始化，首先进行初始化CH378和CS43131,然后从CH378中读取部分可以自定义持久化存储的配置，更新全局的配置状态，完成后设置系统初始化标志位1。两核虽然各自完成初始化，但是设置状态会进行同步。

后续任务规划，V3F会负责处理屏幕模块和配件、供电、键盘之间的交互任务，V5F主要处理屏幕模块和CH378、CS43131、CH585F之间的交互任务，也就是读取文件并给屏幕模块显示、读取音频并播放并同步播放状态给屏幕、蓝牙传输一些信号或者OTA。

两核均等待系统初始化标志位设置完毕之后继续后续任务，由于屏幕模块自主处理大部分任务，小部分需要和系统交互的任务会发回到V5F进行处理。

需要进行core model的架构设计。

文件名统一采用下划线加小写字母的方式，函数名也采用下划线写法仅第一个字母大写，变量名统一小写下划线，文件夹名称采用下划线写法仅第一个字母大写

Common下Core、Debug、Peripheral、Startup不动，作为系统文件，Common/Common/下设置分代码的内容，主要包含hardware这个全局调度的文件，以及CH378、CS43131、键盘、供电、配件相关的文件以及一些基础的驱动文件夹。
