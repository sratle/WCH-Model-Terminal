# Keyboard-3 模块（电子琴）

## 概述

本模块为音乐演奏输入方案，提供琴键、旋钮及控制按键，用于终端内置电子琴 / 合成器应用，**无 CH9329，不支持独立 USB HID 输出**。

- **主控芯片**：CH32V103C8T6
- **布局**：16 个琴键 + 2 个旋钮 + 4 个控制按键
- **与核心模块接口**：模块侧 UART1，**115200/8-N-1**
- **模块 ID**：`0x20`（与 Keyboard-1/2 共用键盘模块 ID，通过子类型区分）
- **模块类型编号**：`0x04`
- **子类型编号**：`0x03`（Music Keyboard）

## 硬件组成

- **CH32V103C8T6**：琴键扫描、旋钮 ADC 采集、控制按键检测、协议帧打包
- **琴键**：16 个按键，映射为半音或全音阶（如 C3~E4）
- **旋钮**：2 路旋转编码器或电位器，用于调节音色、音量、效果等参数
- **控制按键**：4 个功能键（如八度切换、延音、录制、播放）

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、ADC（若旋钮为电位器）、UART1（115200）
- 初始化琴键矩阵或独立按键 GPIO，配置内部上拉
- 初始化旋钮：若使用旋转编码器，配置外部中断或定时器采样；若使用电位器，配置 ADC 通道
- 加载默认音色映射与旋钮功能分配

### 运行阶段

- **琴键扫描**：定时扫描 16 个琴键，检测按下 / 释放事件，支持多键同时按下（和弦）
- **旋钮采集**：
  - 旋转编码器：中断或查询方式读取 A/B 相位，计算旋转方向与步数
  - 电位器：ADC 采样并做平滑滤波（如滑动平均），减少旋钮抖动
- **控制按键检测**：检测 4 个功能键的短按 / 长按事件
- **事件上报**：通过 UART1 向 Core 上报：
  - 琴键按下 / 释放（含音符编号、力度，若支持力度感应）
  - 旋钮旋转事件（含旋钮 ID、旋转方向 / 当前值）
  - 控制按键事件（含按键 ID、事件类型）
- **本地音色切换（可选）**：若模块自带简易合成器，可本地生成音频 PWM 输出到板载蜂鸣器或音频接口；否则所有音频合成由 Core 侧的 CS43131 完成

### 中断与事件

- 定时器中断：琴键扫描周期（建议 2~5 ms，确保快速演奏不漏键）
- ADC / 编码器中断：旋钮采样
- UART1 中断：接收 Core 下发的配置（如音色切换、八度范围）

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`piano.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Piano_Scan()`
- **变量**：统一小写加下划线，例如 `note_state`、`knob_value`
- **文件夹**：下划线写法，仅首字母大写，例如 `Piano`、`Knob`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **无 CH9329**：本模块仅通过 UART1 与 Core 通信，不能独立接电脑当 MIDI 键盘用。若未来有需求，可预留 CH9329 焊盘或增加 USB-MIDI 芯片作为硬件升级。
2. **和弦检测**：16 键需支持多键同时按下，建议采用独立 GPIO 或带二极管的矩阵扫描，避免鬼键；上报时打包多个音符到同一帧或分帧发送。
3. **旋钮精度**：若用于实时调节音色参数（如滤波器截止频率），编码器分辨率建议不低于 20 PPR，且软件做 2 倍或 4 倍频处理；电位器则需保证 ADC 采样稳定。
4. **力度感应（可选）**：若琴键使用压敏或两段式机械开关，可检测按下速度映射为 MIDI 力度值（0~127），提升演奏表现力。
5. **与 Core 的音频闭环**：键盘模块只负责输入，音频合成与播放由 Core 侧的 CS43131 完成；两者需约定好音符编号体系（建议直接采用 MIDI  note number，如 60=C4），降低 Core 侧解析复杂度。
