# Display-2 模块（墨水屏）

## 概述

本模块为低功耗电子墨水屏显示方案，适用于长续航阅读场景，与 Display-1 共用同一套 MiniUI 框架与通信协议。

- **主控芯片**：CH32V307RCT6
- **屏幕规格**：黑白墨水屏，方案待定
- **驱动方式**：SPI 接口
- **与核心模块接口**：模块侧 UART1，**921600/8-N-1**
- **模块 ID**：`0x10`（与 Display-1 共用显示模块 ID，通过子类型区分）
- **模块类型编号**：`0x01`
- **子类型编号**：`0x02`（E-ink）

## 硬件组成

- **CH32V307RCT6**：负责 UI 渲染、UART 协议解析、墨水屏驱动
- **墨水屏面板**：待定型，候选方案：
  - 方案 A：4.26 寸 800×480 触摸黑白墨水屏
  - 方案 B：648×480 非触摸黑白墨水屏
- **SPI 驱动**：CH32V307 通过 SPI 向墨水屏发送刷新命令与图像数据
- **触摸芯片（如方案 A）**：电容 / 电阻触摸控制器

## 软件职责

### 初始化阶段

- 初始化 CH32V307 时钟、GPIO、SPI、UART1（921600）
- 初始化墨水屏驱动：配置分辨率、刷新模式（全刷 / 局部刷新）、LUT 波形
- 初始化 MiniUI 框架（精简版，适配黑白及低刷新特性）
- 初始化 UART 接收状态机，准备接收 Core 指令

### 运行阶段

- **UI 渲染**：执行 MiniUI 绘制，生成 1bpp 帧缓冲，通过 SPI 发送至墨水屏
- **刷新策略**：支持全屏刷新与局部刷新；局部刷新适用于文本阅读、状态更新，全刷用于页面切换以减少残影
- **指令解析**：UART 中断接收 Core 协议帧，执行页面切换、状态栏更新、电子书内容推送等
- **事件上报**：触摸事件（若支持）、屏幕唤醒 / 休眠事件通过 UART 上报 Core

### 中断与事件

- UART1 中断：协议帧接收状态机
- SPI / DMA 中断：图像数据传输完成
- 触摸中断（若支持）：坐标采集，打包为 `CMD_DISP_TOUCH_EVT` 上报

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`eink.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Eink_Refresh()`
- **变量**：统一小写加下划线，例如 `frame_buffer`、`refresh_mode`
- **文件夹**：下划线写法，仅首字母大写，例如 `MiniUI`、`Eink`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **技术方案未确定**：墨水屏尺寸、分辨率、触摸有无均待定，建议在 PCB 设计时预留兼容接口（SPI + 可选 I2C 触摸），避免方案变更导致硬件重制。
2. **刷新速率**：墨水屏刷新周期较长（百毫秒级），MiniUI 需适配异步刷新模型：接收指令 → 计算差分 → 启动刷新 → 等待完成，期间继续接收下一帧数据但做缓冲合并。
3. **局部刷新限制**：并非所有墨水屏面板都支持高质量局部刷新，需根据最终选型的驱动 IC（如 SSD16xx、UC8xxx）评估残影与 ghosting 问题。
4. **低功耗策略**：墨水屏静态显示不耗电，CH32V307 可在刷新间隙进入低功耗模式，仅保留 UART 唤醒中断。
5. **与 Display-1 的协议兼容性**：两者共用 `0x10` 模块 ID 和同一套 Display 协议，Core 侧通过 `CMD_GET_TYPE` 的子类型（`0x01` LCD / `0x02` E-ink）区分，后续 UI 指令逻辑保持一致。
