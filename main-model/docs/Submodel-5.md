# Submodel-5 模块（RGB 点阵）

## 概述

本模块为灯光效果扩展方案，通过 WS2812 可寻址 RGB LED 实现多种动态灯效，用于状态指示、氛围营造或音乐可视化。

- **主控芯片**：CH32V103C8T6
- **与核心模块接口**：模块侧 UART1，**115200/8-N-1**
- **模块 ID**：`0x40` ~ `0x42`
- **模块类型编号**：`0x05`
- **子类型编号**：`0x05`（RGB Lights）

## 硬件组成

- **CH32V103C8T6**：灯效算法运行、WS2812 时序控制、协议帧解析
- **WS2812(B) 灯带 / 点阵**：单线归零码协议，每颗 LED 24bit 颜色数据（GRB 顺序），支持级联
- **布局形式**：
  - 线性灯带（如 8~16 颗围成一圈）
  - 矩形点阵（如 8×8 或 16×16）
- **供电注意**：WS2812 全白时单颗约 60mA，需评估总电流是否超出模块供电能力，必要时外接独立 5V 供电走线

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、SPI/DMA（或 bit-banging）、UART1（115200）
- 初始化 WS2812：将所有 LED 置为全灭状态
- 加载默认灯效参数（亮度、速度、当前模式）
- 向 Core 回复 `CMD_GET_TYPE`，上报子类型 `0x05`

### 运行阶段

- **灯效渲染**：在主循环或定时器中按当前模式计算每帧各 LED 的颜色值：
  - 静态模式：固定颜色、渐变、呼吸
  - 动态模式：彩虹循环、跑马灯、流水、彗星拖尾
  - 音乐可视化（可选）：接收 Core 下发的频谱数据，做柱状图或波形灯效
  - 交互模式：跟随触摸、随电池电量变色等
- **WS2812 刷新**：将计算好的 GRB 数据通过 SPI+DMA（推荐，无需关中断）或精确延时 bit-banging 发送给灯带；刷新频率建议不低于 30 Hz，避免肉眼闪烁
- **指令响应**：
  - 响应 Core 的灯效切换指令：模式 ID + 颜色 + 亮度 + 速度
  - 响应单灯控制指令：指定索引 LED 的颜色（用于精细化指示）
  - 响应开关指令：全灭或恢复记忆状态

### 中断与事件

- 定时器中断：灯效动画帧节拍（如 30~60 Hz）
- SPI / DMA 中断：WS2812 数据发送完成
- UART1 中断：接收 Core 的灯效控制指令

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`ws2812.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Effect_Rainbow()`
- **变量**：统一小写加下划线，例如 `led_count`、`brightness`
- **文件夹**：下划线写法，仅首字母大写，例如 `Ws2812`、`Effect`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **WS2812 时序精确性**：CH32V103 主频 72 MHz，若用 bit-banging，需确保 0/1 码的高低电平持续时间误差 < 150 ns；更推荐使用 SPI 以 2.4 MHz 时钟模拟 WS2812 时序（3bit SPI 数据模拟 1bit WS2812 数据），配合 DMA 实现零 CPU 占用刷新。
2. **功耗与供电**：若灯珠数量多（如 64 颗以上），5V 总电流可达数安培，模块接口的 5V 供电能力需提前评估；必要时灯带独立供电，仅共地。
3. **Gamma 校正**：WS2812 亮度与人眼感知非线性，直接线性调节 RGB 值会导致暗部跳变明显；建议对输入亮度做 Gamma 2.2 校正后再映射到 GRB 数据。
4. **灯效参数平滑切换**：模式切换时避免瞬间跳变，建议在两帧之间做颜色插值（如 HSV 空间线性过渡），提升视觉流畅度。
5. **与 Core 的协同**：音乐可视化模式需要 Core 侧做 FFT 频谱分析后将频点能量数据下发；Submodel-5 仅负责将能量值映射到 LED 高度 / 颜色，不做音频处理。
