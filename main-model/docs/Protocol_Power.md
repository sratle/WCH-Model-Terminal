# Power 模块通信协议

## 1. 模块概述

- **模块类型编号**：`0x03`
- **模块 ID**：`0x30`（预留范围 `0x30~0x31`）

Power 模块为整机提供电源管理功能，硬件特性：

| 特性 | 参数 |
|------|------|
| 电池容量 | 6000 mAh |
| 输出接口 | 5V 系统供电 + 充电宝慢充输出 |
| 充电方式 | PD 快充 / 无线充电 / 慢充 |
| 主控 | CH32V103C8T6 |

### 1.1 帧格式速查

Power 模块通过 **UART5** 与 Core（V3F）通信，物理层参数：

| 参数 | 值 |
|------|-----|
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 流控 | 无 |

**帧结构**（总长度最大 264 字节）：

```
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
|  HEAD  |  SRC   |  DST   |  LEN   |  CMD   |  DATA[0..N] | TAIL0  | TAIL1  | TAIL2  | TAIL3  |
| 1 byte | 1 byte | 1 byte | 1 byte | 1 byte |   N bytes   | 1 byte | 1 byte | 1 byte | 1 byte |
+--------+--------+--------+--------+--------+-------------+--------+--------+--------+--------+
```

- **HEAD**：固定 `0xAA`。仅在接收状态机处于“等待帧头”时识别为新帧起始，数据域中的 `0xAA` 不会触发重解析。
- **LEN**：`CMD + DATA` 的总字节数，即 `N = LEN - 1`，范围 `1 ~ 255`。`LEN = 1` 表示 DATA 为空。
- **DATA**：数据域，长度由 LEN 推导，最大 255 字节。
- **TAIL**：固定序列 `A5 5A FC FD`，任一字节不匹配则丢弃整帧。

**常用模块 ID**：

| ID | 模块 |
|----|------|
| `0x00` | Core（主控） |
| `0x30` | Power（本模块） |

**帧长度计算**：
```
总帧长 = 1(HEAD) + 1(SRC) + 1(DST) + 1(LEN) + 1(CMD) + N(DATA) + 4(TAIL)
       = 9 + N
       = 8 + LEN
```

> 完整协议规范（含接收状态机、通用操作码 `0x00~0x0F`、NACK 错误码定义）见 `PROJECT.md`。

## 2. 操作码定义

Power 模块专用操作码范围为 `0x31 ~ 0x3F`，按模块 ID 高 4 位 `0x3` 统一分配。

### 2.1 基础操作码（0x31 ~ 0x3F）

| 操作码 | 宏名 | 方向 | 说明 | DATA 格式 |
| --- | --- | --- | --- | --- |
| `0x31` | `CMD_PWR_GET_STATUS` | Core -> Power | 查询当前电源完整状态 | 无，Power 回复 ACK + `[状态位图:1][电量:1][电压:2][电流:2][温度:1][功率:2]` |
| `0x32` | `CMD_PWR_GET_BATTERY_INFO` | Core -> Power | 查询电池详细信息 | 无，Power 回复 ACK + `[设计容量:2][实际容量:2][循环次数:2][健康度:1][类型:1]` |
| `0x33` | `CMD_PWR_SET_CHARGE_POLICY` | Core -> Power | 设置充电策略 | `[策略:1]` |
| `0x34` | `CMD_PWR_SET_OUTPUT_POLICY` | Core -> Power | 设置充电宝输出策略 | `[开关:1][限制功率:2]` |
| `0x35` | `CMD_PWR_SET_ALARM_THRESHOLD` | Core -> Power | 设置告警阈值 | `[低电量阈值:1][过温阈值:1][关机阈值:1]` |
| `0x36` | `CMD_PWR_STATUS_REPORT` | Power -> Core | 电源状态周期性/变化上报 | `[状态位图:1][电量:1][电压:2][电流:2][温度:1][功率:2]` |
| `0x37` | `CMD_PWR_CHARGE_EVENT` | Power -> Core | 充电状态变化事件 | `[事件类型:1][功率:2]` |
| `0x38` | `CMD_PWR_ALARM_EVENT` | Power -> Core | 告警事件（低电量/过温/过流） | `[告警类型:1][当前值:2]` |
| `0x39~0x3F` | — | — | 预留 | — |

## 3. ACK/NACK 响应规则

### 3.1 设计原则

ACK/NACK 由**命令语义**决定：

| 语义类别 | 说明 | 响应要求 |
|----------|------|---------|
| **查询 (Query)** | 请求获取某信息 | **必须 ACK + 数据** |
| **设置 (Set)** | 修改某参数/策略 | **发送即忘，不响应** |
| **事件 (Event)** | 单向状态/告警上报 | **不需要响应** |

> **发送即忘**：Core 向 Power 发送设置类命令后不需要等待 ACK。Power 被动执行，不支持时静默丢弃。

### 3.2 操作码响应要求

| 码 | 命令 | 语义 | 接收方行为 |
|---|---|---|---|
| `0x31` | GET_STATUS | 查询 | Power 回复 ACK + 9 字节状态数据 |
| `0x32` | GET_BATTERY_INFO | 查询 | Power 回复 ACK + 8 字节电池信息 |
| `0x33` | SET_CHARGE_POLICY | 设置 | 发送即忘，Power 不回复 |
| `0x34` | SET_OUTPUT_POLICY | 设置 | 发送即忘，Power 不回复 |
| `0x35` | SET_ALARM_THRESHOLD | 设置 | 发送即忘，Power 不回复 |
| `0x36` | STATUS_REPORT | 事件 | Core 不回复 |
| `0x37` | CHARGE_EVENT | 事件 | Core 不回复 |
| `0x38` | ALARM_EVENT | 事件 | Core 不回复 |

### 3.3 ACK / NACK 帧格式

**ACK 帧**：
```
[AA][DST][SRC][LEN][CMD=0x04][DATA...][A5][5A][FC][FD]
```
- SRC/DST 与原请求**交换**
- DATA 可选携带响应数据，`LEN = 1 + data_len`
- 空 ACK：`LEN = 01`，无 DATA

**NACK 帧**：
```
[AA][DST][SRC][02][CMD=0x05][错误码:1][A5][5A][FC][FD]
```
- DATA[0] = 错误码（1 字节），详见 `PROJECT.md` 通用协议错误码定义
- `LEN = 02`

### 3.4 Core 侧超时处理

Core 发送**查询类**命令后，应在 **500ms** 内收到 ACK/NACK。超时后由应用层处理。**协议层不实现自动重传**。

## 4. 数据格式详解

### 4.1 电源状态格式（CMD 0x31 / 0x36）

**查询响应（ACK，Core → Power，0x31）** 与 **状态上报（Power → Core，0x36）** 使用相同的 DATA 格式：

```
DATA[0]: 状态位图（1字节）
  - bit0: 是否充电中（0=否, 1=是）
  - bit1: 是否已充满（0=否, 1=是）
  - bit2: 是否充电宝输出中（0=否, 1=是）
  - bit3: 电池健康度告警（0=正常, 1=健康度低于阈值）
  - bit4~7: 预留
DATA[1]: 电量百分比（1字节，0~100）
DATA[2..3]: 电池电压（uint16 大端，单位 mV）
DATA[4..5]: 电池电流（int16 大端，单位 mA，正值=充电，负值=放电）
DATA[6]: 电池温度（int8，单位 °C，范围 -40~85）
DATA[7..8]: 供电/充电功率（uint16 大端，单位 mW）
```

> 总 DATA 长度：9 字节，`LEN = 10`。

**示例**：电量 67%，电压 4200mV，充电中 1200mA，温度 35°C，充电功率 5040mW
```
DATA[0] = 0x01  /* bit0=1: 充电中 */
DATA[1] = 0x43  /* 电量 = 67% */
DATA[2] = 0x10  /* 电压高字节 */
DATA[3] = 0x68  /* 电压低字节 = 4200mV */
DATA[4] = 0x04  /* 电流高字节 */
DATA[5] = 0xB0  /* 电流低字节 = 1200mA（正值=充电） */
DATA[6] = 0x23  /* 温度 = 35°C */
DATA[7] = 0x13  /* 功率高字节 */
DATA[8] = 0xB0  /* 功率低字节 = 5040mW */
```

### 4.2 电池信息格式（CMD 0x32）

**查询响应（ACK）**：

```
DATA[0..1]: 电池设计容量（uint16 大端，单位 mAh，如 6000）
DATA[2..3]: 当前实际容量（uint16 大端，单位 mAh）
DATA[4..5]: 循环次数（uint16 大端）
DATA[6]: 电池健康度（1字节，0~100，百分比）
DATA[7]: 电池类型（1字节）
  - 0x00: 锂离子    - 0x01: 锂聚合物    - 0x02: 固态电池
```

> 总 DATA 长度：8 字节，`LEN = 9`。

### 4.3 充电策略格式（CMD 0x33）

**设置充电策略（Core → Power）**：

```
DATA[0]: 充电策略（1字节）
  - 0x00: 关闭充电
  - 0x01: 慢充（5V/1A 或更低）
  - 0x02: PD 快充
  - 0x03: 无线充电
  - 0x04: 智能模式（Power 模块根据温度和电量自主决定）
  - 0x80~0xFF: 用户自定义策略
```

> 总 DATA 长度：1 字节，`LEN = 2`。

### 4.4 输出策略格式（CMD 0x34）

**设置充电宝输出策略（Core → Power）**：

```
DATA[0]: 输出开关（1字节）
  - 0x00: 关闭充电宝输出
  - 0x01: 开启充电宝输出
DATA[1..2]: 输出限制功率（uint16 大端，单位 mW）
  - 0x0000: 使用默认限制
  - 0xFFFF: 不限制
```

> 总 DATA 长度：3 字节，`LEN = 4`。

### 4.5 告警阈值格式（CMD 0x35）

**设置告警阈值（Core → Power）**：

```
DATA[0]: 低电量告警阈值（1字节，0~100，默认 20%）
  - 电量低于此值时触发 ALARM_EVENT（类型=0x00）
DATA[1]: 过温告警阈值（1字节，°C，默认 60°C）
  - 温度高于此值时触发 ALARM_EVENT（类型=0x02）
DATA[2]: 低电量关机阈值（1字节，0~100，默认 5%）
  - 电量低于此值时 Power 模块自动关机，并触发 ALARM_EVENT（类型=0x01）
```

> 总 DATA 长度：3 字节，`LEN = 4`。

### 4.6 充电事件格式（CMD 0x37）

**充电状态变化上报（Power → Core）**：

```
DATA[0]: 事件类型（1字节）
  - 0x00: 充电器插入
  - 0x01: 充电器拔出
  - 0x02: 开始充电
  - 0x03: 充电完成（已充满）
  - 0x04: 切换为快充模式
  - 0x05: 切换为慢充模式
  - 0x06: 切换为无线充电模式
  - 0x07: 充电异常停止（过温/过流保护）
DATA[1..2]: 当前充电功率（uint16 大端，单位 mW，事件发生时）
  - 充电器拔出时为 0
```

> 总 DATA 长度：3 字节，`LEN = 4`。

### 4.7 告警事件格式（CMD 0x38）

**告警上报（Power → Core）**：

```
DATA[0]: 告警类型（1字节）
  - 0x00: 低电量告警（电量低于低电量阈值）
  - 0x01: 电量危急（电量低于关机阈值，即将自动关机）
  - 0x02: 过温告警（温度高于过温阈值）
  - 0x03: 过流告警（充电/放电电流超过安全范围）
  - 0x04: 电池健康度低（健康度低于 50%）
DATA[1..2]: 当前值（uint16 大端）
  - 低电量/危急: 当前电量百分比（0~100）
  - 过温: 当前温度（°C，带符号，范围 -40~85，以 uint16 表示时 +40 偏移）
  - 过流: 当前电流绝对值（mA）
  - 健康度低: 当前健康度百分比（0~100）
```

> 总 DATA 长度：3 字节，`LEN = 4`。

> **温度偏移说明**：告警事件中温度采用 `uint16` 存储，实际值 = 存储值 - 40。例如 75°C 存储为 115（0x0073），-10°C 存储为 30（0x001E）。

## 5. CMD_GET_TYPE 响应

Power 模块响应 `CMD_GET_TYPE` 时，`CMD_ACK` 的 DATA 格式如下：

| 字节 | 字段 | 值/说明 |
|------|------|---------|
| DATA[0] | 模块类型编号 | `0x03` |
| DATA[1] | 模块子类型编号 | `0x01`（`MODULE_SUBTYPE_POWER_STANDARD`） |
| DATA[2] | 硬件版本号 | 电源管理硬件版本 |
| DATA[3] | 固件主版本号 | 电源管理固件主版本 |
| DATA[4] | 固件次版本号 | 电源管理固件次版本 |
| DATA[5..N] | 扩展信息 | 电池类型、最大充电电流、是否支持无线充等能力位（可选） |

**Power 子类型定义**

> 代码宏定义：`protocol.h` → `MODULE_SUBTYPE_POWER_*`

| 子类型编号 | 宏名 | 实现名称 | 说明 |
|------------|------|----------|------|
| `0x00` | `MODULE_SUBTYPE_POWER_RESERVED` | 保留 | 系统保留 |
| `0x01` | `MODULE_SUBTYPE_POWER_STANDARD` | 标准供电 | 当前唯一实现（6000mAh + PD + 无线充 + 充电宝） |
| `0x02~0xFF` | — | 预留 | 未来扩展 |
