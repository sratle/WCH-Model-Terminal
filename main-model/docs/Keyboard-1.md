# Keyboard-1 模块（主键盘）

## 概述

本模块为标准配列键盘输入方案，既作为移动终端的主输入设备，也可独立作为 USB HID 设备连接电脑使用。

- **主控芯片**：CH32V103C8T6
- **配列**：约 40% 配列（自定义键位）
- **与核心模块接口**：模块侧 UART1，**115200/8-N-1**
- **模块 ID**：`0x20`
- **模块类型编号**：`0x04`
- **子类型编号**：`0x01`（Main Keyboard）

## 硬件组成

- **CH32V103C8T6**：矩阵键盘扫描、协议帧打包、CH9329 通信
- **键矩阵**：约 40% 配列机械按键矩阵
- **CH9329**：串口转 HID 芯片，将键码转换为标准 USB HID 键盘报告
- **USB-A 接口**：Device 口，输出 HID 信号，可接 PC 等主机

## 软件职责

### 初始化阶段

- 初始化 CH32V103 时钟、GPIO、UART1（115200）、UART2（接 CH9329，波特率按 CH9329 要求配置，通常为 9600 或 115200）
- 初始化键矩阵 GPIO：配置行输出 / 列输入，使能内部上拉
- 初始化 CH9329：发送初始化命令，设置为标准键盘模式
- 加载当前键位布局（如有持久化需求）

### 运行阶段

- **矩阵扫描**：定时扫描键矩阵，检测按键按下 / 释放 / 长按事件，进行硬件消抖
- **键码映射**：将物理行列坐标映射为逻辑键码（支持多层配列与 Fn 组合键）
- **双通道输出**：
  - 通过 **UART1** 向 Core 上报键码（`CMD_KBD_KEY_DOWN` / `CMD_KBD_KEY_UP` / `CMD_KBD_FN_COMBO`）
  - 通过 **UART2 → CH9329 → USB-A** 输出标准 HID 键盘报告，实现接电脑即插即用
- **背光控制（如有）**：接收 Core 的 `CMD_KBD_SET_BACKLIGHT` 指令，控制键盘背光亮度或灯效模式

### 中断与事件

- 定时器中断：矩阵扫描周期（建议 1~5 ms）
- UART1 中断：接收 Core 下发的配置或背光指令
- UART2 中断：接收 CH9329 状态反馈（如连接状态变化）

## 命名规范

- **文件**：统一采用下划线加小写字母，例如 `hardware.c`、`ch9329.h`
- **函数**：每个单词首字母大写加下划线，例如 `Hardware_Init()`、`Matrix_Scan()`
- **变量**：统一小写加下划线，例如 `key_state`、`fn_pressed`
- **文件夹**：下划线写法，仅首字母大写，例如 `Keyboard`、`CH9329`
- **系统目录**（`Core`、`Debug`、`Peripheral`、`Startup`）保持不动，不参与上述命名规范

## 开发要点

1. **CH9329 波特率**：需查阅 CH9329 数据手册确认默认波特率，通常支持 9600 / 115200，上电后可能需要发送特定配置帧设定工作模式。
2. **双通道输出一致性**：同一按键事件需同时向 Core（自定义协议）和 CH9329（HID 报告）输出，两者键码映射表独立维护；接电脑时无需依赖 Core 即可工作。
3. **Fn 组合键**：建议在键盘本地处理 Fn 层切换，仅将最终生效键码上报 Core，避免 Core 侧再做一层 Fn 解析。
4. **消抖与 N-KRO**：CH32V103 资源有限，建议采用基本的软件消抖（如 5~10 ms 延时判稳）；若需全键无冲，需优化矩阵扫描算法或采用移位寄存器扩展。
5. **布局查询**：支持 Core 通过 `CMD_KBD_GET_LAYOUT` 查询当前键位布局，便于 Core 侧 UI 显示虚拟键盘或快捷键提示。
